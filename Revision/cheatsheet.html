📍 Week 1 — KNN & Feature Scaling

Distance Metrics

Euclidean Distance d = √[ (p₁−q₁)² + (p₂−q₂)² + … + (pₙ−qₙ)² ] Straight-line distance. Default for KNN.
Manhattan Distance d = |p₁−q₁| + |p₂−q₂| + … + |pₙ−qₙ| Sum of absolute differences. More robust to outliers.
Minkowski Distance (general) d = ( |p₁−q₁|ʳ + |p₂−q₂|ʳ + … )^(1/r) r=1 → Manhattan · r=2 → Euclidean
  • KNN → majority vote of K nearest (classification) or mean (regression)
  • Small K → overfits (jagged boundary) · Large K → underfits
  • Rule of thumb: K ≈ √N · Use odd K to avoid ties
  • ⚠️ Always scale features before KNN

Feature Scaling

Min-Max Normalisation → range [0, 1] x' = (x − x_min) / (x_max − x_min)
Z-Score Standardisation → mean=0, std=1 x' = (x − μ) / σ μ = mean of feature · σ = standard deviation

Gradient Descent

Parameter Update Rule θ := θ − α × ∂J/∂θ α = learning rate · ∂J/∂θ = gradient of cost w.r.t. θ
  • Too large α → overshoots, may diverge
  • Too small α → converges very slowly
  • Update ALL θⱼ simultaneously using old values
📈 Week 2 — Regression & Regularization ✅ Linear Regression in Final 🚫 Poly Reg & Ridge/Lasso NOT in Final

Linear Regression

Hypothesis (Prediction) ŷ = θ₀ + θ₁x₁ + θ₂x₂ + … + θₙxₙ
MSE Cost Function J(θ) = (1/2m) × Σᵢ (ŷᵢ − yᵢ)² m = number of training examples · Sum over all i
Normal Equation (closed-form) θ = (XᵀX)⁻¹ Xᵀy No iterations needed · Slow for many features (O(n³))
GD Gradient for θ₀ ∂J/∂θ₀ = (1/m) × Σ (ŷᵢ − yᵢ)
GD Gradient for θ₁ ∂J/∂θ₁ = (1/m) × Σ (ŷᵢ − yᵢ) × xᵢ
  • MSE is convex → gradient descent always finds global min
  • GD preferred when n (features) > 10,000

Bias-Variance Tradeoff

Total Error Decomposition Total Error = Bias² + Variance + σ² (noise) σ² = irreducible noise — cannot be eliminated
  • High Bias = Underfitting — high train AND test error, model too simple
  • High Variance = Overfitting — low train error, HIGH test error, memorises noise
  • More data helps variance, NOT bias
  • Regularisation reduces variance
Gradient Descent vs Normal Equation GD: needs α · iterative · scales to large n Normal: no α · one-step · slow for large n

Regularization 🚫 NOT IN FINAL

Ridge L2 — cost + penalty J(θ) + λ × (θ₁² + θ₂² + … + θₙ²) Shrinks weights toward 0 but never exactly 0
Lasso L1 — cost + penalty J(θ) + λ × (|θ₁| + |θ₂| + … + |θₙ|) Can set weights to exactly 0 → feature selection

✂️ Week 3–4 — Support Vector Machines 🚫 EXCLUDED FROM FINAL

Hard Margin SVM

Decision boundary w · x + b = 0
Margin width margin = 2 / ‖w‖
Objective — minimise ½ ‖w‖² subject to: yᵢ(w·xᵢ + b) ≥ 1
  • Predict: sign(w·x + b)
  • Support vectors: yᵢ(w·xᵢ + b) = 1 exactly

Soft Margin (C-SVM)

Objective — minimise ½‖w‖² + C × Σ ξᵢ
Constraint yᵢ(w·xᵢ + b) ≥ 1 − ξᵢ, ξᵢ ≥ 0
  • ξᵢ = 0 → correct, outside margin
  • 0 < ξᵢ < 1 → inside margin, correct side
  • ξᵢ ≥ 1 → misclassified
  • Large C → small margin (→ overfit)
  • Small C → large margin (→ underfit)

Kernels

LinearK(x, z) = xᵀz
PolynomialK(x, z) = (xᵀz + c)^d
RBF / GaussianK(x, z) = exp(−γ ‖x − z‖²)
  • RBF → maps to infinite-dimensional space
  • Large γ → narrow, overfits · Small γ → underfits
🌳 Week 5 — Decision Trees 🚫 EXCLUDED FROM FINAL

Impurity Measures

Entropy (Shannon) H(S) = − Σ pₖ × log₂(pₖ) Pure node → H=0 · 50/50 split → H=1 bit
Information Gain (ID3 criterion) IG(S, A) = H(S) − Σᵥ (|Sᵥ|/|S|) × H(Sᵥ)
Gain Ratio (C4.5 — fixes IG bias) GainRatio = IG(S,A) / SplitInfo(S,A)
Gini Index (CART / sklearn) Gini = 1 − Σ pₖ²
  • ID3 → max IG · C4.5 → Gain Ratio · sklearn → Gini
  • Boundaries: always axis-parallel rectangles

Pruning

⚙️ Week 6–7 — Clustering

K-Means Algorithm

Objective — Inertia (WCSS) J = Σₖ Σ(xᵢ in Cₖ) ‖xᵢ − μₖ‖²
Assignment step cᵢ = argmin_k ‖xᵢ − μₖ‖² Assign each point to the nearest centroid
Update step μₖ = (1/|Cₖ|) × Σ(xᵢ in Cₖ) xᵢ New centroid = arithmetic mean of all assigned points
  • Converges to local optimum (non-convex J)
  • More K → always lower inertia — use Elbow method
  • ⚠️ Fails on concentric rings / elongated shapes
  • ⚠️ Always standardise features first
Exam: Concentric rings → K-Means fails because it minimises Euclidean distance and assumes spherical clusters. Inner and outer ring centroids overlap.

K-Means++ & X-Means

K-Means++ — initialisation probability P(x chosen) ∝ d(x)² d(x) = distance to nearest already-chosen centroid
  • Picks far-apart initial centroids → avoids bad local optima
  • Default in sklearn: init='k-means++'
X-Means — auto-selects K using BIC BIC = −2 × ln(L̂) + p × ln(n) Lower BIC = better model · Split cluster if BIC improves
  • Starts at K=2, splits clusters if BIC improves
  • No need to specify K upfront

Distance Metrics for Clustering

  • Euclidean: straight-line, scale-sensitive, default
  • Manhattan: sum of absolute diffs, more outlier-robust
  • Cosine: measures angle not magnitude → good for text/NLP

Hierarchical (Agglomerative)

  • Start: each point = own cluster
  • Repeat: merge two closest clusters
  • Result: dendrogram — cut at height → K clusters
  • No need to specify K upfront ✅
LinkageDistance = ?Good for
Singlemin distance any pairElongated, rings ✅
Completemax distance any pairCompact, spherical
Averagemean of all pairsBalanced
Exam Q2 (Assignment mirror): For concentric rings → use single-linkage. Complete-linkage uses the maximum distance between cluster members — the far side of the outer ring is very far from the inner ring → merges fail. Single-linkage only needs one close pair → can chain along the ring.
🚨 Week 8 — Anomaly Detection

Isolation Forest

Anomaly Score s(x, n) = 2 ^ (−E[h(x)] / c(n)) E[h(x)] = average path length in random trees
Normalisation factor c(n) c(n) = 2H(n−1) − 2(n−1)/n H = harmonic number H(k) = 1 + 1/2 + 1/3 + … + 1/k
  • Score ≈ 1 → anomaly (isolated early, short path)
  • Score ≈ 0.5 → normal
  • Score ≈ 0 → very normal (buried deep)
  • Anomalies isolated in fewer splits than normal points

Gaussian Model

Product of per-feature Gaussians p(x) = p(x₁; μ₁, σ₁²) × p(x₂; μ₂, σ₂²) × … × p(xₙ; μₙ, σₙ²)
Gaussian PDF for feature j p(xⱼ; μⱼ, σⱼ²) = (1 / √(2πσⱼ²)) × exp(−(xⱼ−μⱼ)² / 2σⱼ²)
Decision rule Flag as anomaly if p(x) < ε (threshold)
  • μⱼ = mean of feature j · σⱼ² = variance of feature j
  • Use anomaly detection (not supervised) when positives are rare (0–20 examples) and future anomalies may look different from past ones

Confusion Matrix

Actual +TPFN

Accuracy Accuracy = (TP + TN) / (TP + TN + FP + FN)
Accuracy fails on imbalanced data. If 9,950 negatives and 50 positives — predicting all negative gives 99.5% accuracy but 0% recall. Useless!
Classic Exam Scenario (Assignment Q1) 10,000 transactions · 50 fraud · predict all negative: TP=0 · FP=0 · FN=50 · TN=9950 Accuracy = 9950/10000 = 99.5% ← misleading Recall = 0/50 = 0% ← catches zero fraud

Precision, Recall, F1

Precision — "how exact are my positive predictions?" Precision = TP / (TP + FP)
Recall (Sensitivity) — "how many positives did I catch?" Recall = TP / (TP + FN)
F1-Score — harmonic mean of Precision & Recall F1 = 2 × (Precision × Recall) / (Precision + Recall) = 2×TP / (2×TP + FP + FN) F1 = 0 if either Precision or Recall is 0
  • Use F1 when classes are imbalanced
  • Precision↑ → fewer false alarms (FP)
  • Recall↑ → fewer missed positives (FN)

SMOTE & Cost-Sensitive Learning

SMOTE synthetic sample formula x_new = xᵢ + λ × (x_neighbour − xᵢ) λ ∈ [0, 1] random · x_neighbour = KNN within minority class
  • Interpolates between two minority class neighbours in feature space
  • Creates new synthetic minority points along the line segment
  • Not random noise — stays in the minority feature region

Data Leakage ⭐ Most Tested

Definition: Including target-variable information in training data that won't be available at prediction time.

Type 1 — Train-Test Contamination: Fitting scaler/encoder on full dataset before splitting → test statistics contaminate training.

Type 2 — Target Leakage: Using features only available after the event (e.g., post-diagnosis data to predict a diagnosis).

✅ Correct pipeline order 1. Split data into train / test FIRST 2. Fit StandardScaler on TRAIN only 3. Transform (apply) scaler to both train and test 4. Fit TargetEncoder on TRAIN only 5. Transform test with train-fitted encoder 6. Train model on transformed train set

📉 Week 13–14 — Model Drift & KS Test (⭐ High Priority)

Data Drift vs Concept Drift

Data Drift (Covariate Shift) P(X) changes — input distribution shifts P(Y|X) stays same — true relationship unchanged
Concept Drift P(Y|X) changes — the true mapping changes Same inputs now give different correct outputs
Data DriftConcept Drift
ChangesP(X)P(Y|X)
RiskMediumHigh

KS Test — Kolmogorov–Smirnov

KS Statistic D D = max | F_train(x) − F_prod(x) | F = empirical CDF · sup = maximum gap between the two CDFs
D valueStatusAction
D < 0.1No driftKeep monitoring
0.1 ≤ D < 0.2Mild driftMonitor closely
D ≥ 0.2Significant!Investigate, retrain
p < 0.05Statistically sig.Drift confirmed
Lecture example — Reference {1,2,3,4} vs Production {3,4,5,6} At x=3: F_train(3)=0.75, F_prod(3)=0.25 → gap = 0.50 At x=4: F_train(4)=1.00, F_prod(4)=0.50 → gap = 0.50 D = max gap = 0.5 → Significant drift (D ≥ 0.2) ✅

FGSM — Fast Gradient Sign Method x_adversarial = x + ε × sign(∇ₓ J(θ, x, y)) ε = small perturbation · sign(∇ₓJ) = direction of max loss increase
Tabular vs Image adversarial inputs: Image = imperceptible pixel noise. Tabular = crafted feature values within plausible ranges (e.g., manipulate transaction features to evade fraud detection).

Fairness Metrics

Demographic Parity P(Ŷ=1 | Group A) = P(Ŷ=1 | Group B) Same positive prediction rate across all groups
Equal Opportunity TPR_GroupA = TPR_GroupB Qualified candidates have equal chance of positive outcome
Equalized Odds TPR_A = TPR_B AND FPR_A = FPR_B Both true positive AND false positive rates must match
Calibration If model says 70% → should be right 70% of the time for every group Confidence equally reliable across all groups
Key: Fairness metrics can CONFLICT with each other and with accuracy. No single definition is always "fair" — depends on context.
