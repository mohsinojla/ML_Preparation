K-Nearest Neighbours (KNN)

W1 ▼

Core Idea

  • Non-parametric, lazy learner — memorises all training data
  • Predict by majority vote (classification) or mean (regression) of K nearest neighbours
  • No training phase — all cost at prediction time
  • Small K → overfitting (jagged boundary); Large K → underfitting
  • Rule of thumb: $K \approx \sqrt{N}$; always use odd K for binary
Euclidean
$$d(\mathbf{p},\mathbf{q})=\sqrt{\sum_{i=1}^n(p_i-q_i)^2}$$
Manhattan
$$d(\mathbf{p},\mathbf{q})=\sum_{i=1}^n|p_i-q_i|$$
Must Know
Always scale features before KNN — unscaled features with large magnitudes dominate distance.
Exam Trap
KNN is called "lazy" because it doesn't learn a model — NOT because it's slow or inaccurate.
Likely Exam Q
Given 5 training points and K=3, classify a new point: compute all distances, rank, take majority vote.

Feature Scaling:

Min-Max
$$x'=\frac{x-x_{min}}{x_{max}-x_{min}}$$

→ scales to [0,1]

Z-Score
$$x'=\frac{x-\mu}{\sigma}$$

→ mean=0, std=1

Gradient Descent

W1▼
Update Rule
$$\theta := \theta - \alpha \frac{\partial J(\theta)}{\partial \theta}$$
VariantUsesNoise
Batch GDAll dataNone (stable)
SGD1 sampleHigh
Mini-Batchm samplesMedium
Exam Trap
Too large $\alpha$ → diverges (cost increases). Too small → converges very slowly. Both are wrong.
Simultaneous Update
Update ALL $\theta_j$ using OLD values — never use new $\theta_0$ to compute new $\theta_1$.
Exam Trap
For linear regression, MSE cost is convex → gradient descent always finds the global minimum. No local minima.

Linear & Polynomial Regression Poly Reg 🚫 Not in Final

W2▼
Hypothesis
$$h_\theta(\mathbf{x})=\theta_0+\theta_1x_1+\cdots+\theta_nx_n=\boldsymbol{\theta}^T\mathbf{x}$$
MSE Cost
$$J(\theta)=\frac{1}{2m}\sum_{i=1}^m(h_\theta(x^{(i)})-y^{(i)})^2$$
Normal Equation
$$\boldsymbol{\theta}=(\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}$$

Closed-form; $O(n^3)$ — slow for large $n$

🚫 Polynomial Regression — Not in Final Linear Regression is included; Polynomial Regression is excluded.

Polynomial: define $x_2=x^2$, $x_3=x^3$ → still linear regression on new features. (excluded)

Exam Trap
Polynomial regression is still linear in the parameters $\theta$ — just non-linear in the input $x$.
MethodNeeds $\alpha$?Best for
Gradient DescentYesLarge n (features)
Normal EquationNoSmall n (<10k)
Exam Q Type
Given data, do 1 GD step: compute predictions, errors, gradients, then update $\theta_0$ and $\theta_1$.

Bias-Variance Tradeoff

W2▼
Decomposition
$$\text{Expected Error} = \text{Bias}^2 + \text{Variance} + \underbrace{\sigma^2}_{\text{irreducible noise}}$$

High Bias = Underfitting

  • High train error AND high test error
  • Model too simple
  • Fix: more complexity, more features

High Variance = Overfitting

  • Low train error, HIGH test error
  • Model memorises noise
  • Fix: more data, regularisation, simpler model
Exam Trap
"Adding more training data helps with variance (overfitting) but NOT with bias (underfitting)." — This distinction is commonly tested.

Ridge & Lasso Regularization 🚫 Not in Final

W2▼
Ridge (L2)
$$J=\text{MSE}+\lambda\sum_j\theta_j^2$$

Shrinks toward 0, never exactly 0

Lasso (L1)
$$J=\text{MSE}+\lambda\sum_j|\theta_j|$$

Can set weights exactly to 0 → feature selection

Ridge (L2)Lasso (L1)
Zero coefs?No (near 0)Yes exactly
Feature sel.NoYes
Closed formYesNo
Exam Trap
Do NOT regularise $\theta_0$ (bias term). Only $\theta_1\ldots\theta_n$ are penalised. $\lambda\to\infty$: all weights→0, model predicts mean (extreme underfitting).

Support Vector Machines (Hard + Soft Margin) 🚫 Not in Final

W3–W4▼
Decision Boundary
$$\mathbf{w}^T\mathbf{x}+b=0$$

Predict: $\hat{y}=\text{sign}(\mathbf{w}^T\mathbf{x}+b)$

Margin Width
$$\text{margin}=\frac{2}{\|\mathbf{w}\|}$$

Maximise margin ≡ minimise $\frac{1}{2}\|\mathbf{w}\|^2$

Hard Margin Primal
$$\min\frac{1}{2}\|\mathbf{w}\|^2 \quad \text{s.t. } y_i(\mathbf{w}^T\mathbf{x}_i+b)\geq 1$$
Soft Margin (C-SVM)
$$\min\frac{1}{2}\|\mathbf{w}\|^2+C\sum_i\xi_i \quad \text{s.t. } y_i(\mathbf{w}^T\mathbf{x}_i+b)\geq 1-\xi_i$$

Slack Variables $\xi_i$:

  • $\xi_i=0$ → outside margin, correctly classified
  • $0<\xi_i<1$ → inside margin, correct side
  • $\xi_i\geq 1$ → misclassified

C parameter:

  • Large C → small margin, few violations (→ overfitting)
  • Small C → large margin, more violations (→ underfitting)
Exam Trap
Support vectors are points on the margin boundary ($y_i(\mathbf{w}^T\mathbf{x}_i+b)=1$). Remove any non-SV → hyperplane unchanged.
Exam Q
$\mathbf{w}=[3,4]$: margin $= 2/5 = 0.4$. Classify $\mathbf{x}=[3,1]$: compute $\mathbf{w}^T\mathbf{x}+b$, take sign.

Kernel Trick 🚫 Not in Final

W4▼
Core Idea
Replace every inner product $\langle\mathbf{x}_i,\mathbf{x}_j\rangle$ in the dual with $K(\mathbf{x}_i,\mathbf{x}_j)=\langle\phi(\mathbf{x}_i),\phi(\mathbf{x}_j)\rangle$ — implicit high-dim mapping without computing $\phi(\mathbf{x})$ explicitly.
KernelFormulaFeature space
Linear$\mathbf{x}_i^T\mathbf{x}_j$Same
Polynomial$(\mathbf{x}_i^T\mathbf{x}_j+c)^d$Degree-d polys
RBF/Gaussian$\exp(-\gamma\|\mathbf{x}_i-\mathbf{x}_j\|^2)$∞-dimensional
Exam Trap
RBF maps to infinite-dimensional space yet is computed in $O(d)$ time. Large $\gamma$ = narrow Gaussian = overfitting. Small $\gamma$ = underfitting.
Exam Q
Poly kernel $K(\mathbf{x},\mathbf{z})=(\mathbf{x}^T\mathbf{z})^2$, $\mathbf{x}=[1,2]$, $\mathbf{z}=[3,4]$: $=(3+8)^2=121$.

Decision Trees & Pruning 🚫 Not in Final

W5▼
Entropy (K classes)
$$H(S)=-\sum_{k=1}^K p_k\log_2 p_k$$

Pure node: $H=0$ · Max impurity: $H=1$ (binary, 50/50)

Information Gain (ID3)
$$\text{IG}(S,A)=H(S)-\sum_v\frac{|S_v|}{|S|}H(S_v)$$
Gain Ratio (C4.5)
$$\text{GainRatio}=\frac{\text{IG}(S,A)}{\text{SplitInfo}(S,A)}$$

Corrects IG's bias toward many-valued features

Gini Index (CART)
$$\text{Gini}(S)=1-\sum_k p_k^2$$

Decision Boundaries: always axis-parallel rectangles (unlike SVM diagonal hyperplane)

Pruning:

Pre-PruningPost-Pruning
WhenDuring growthAfter full tree
SpeedFastSlower
Risk"Short-sighted""Hindsight" (better)
Cost-Complexity
$$R_\alpha(T)=R(T)+\alpha|T|$$

$|T|$=leaf count, $\alpha$=complexity penalty

Exam Trap
ID3 uses Max-Info-Gain. C4.5 uses Gain Ratio. sklearn (CART) uses Gini by default. Know which is which!

Compute IG for a feature that splits {9+, 5-} into {Weak: 6+2-} and {Strong: 3+3-}

Show Answer
$H(S)=-(9/14)\log_2(9/14)-(5/14)\log_2(5/14)\approx0.940$
$H(S_{weak})=-(6/8)\log_2(6/8)-(2/8)\log_2(2/8)\approx0.811$
$H(S_{strong})=-0.5\log_2(0.5)-0.5\log_2(0.5)=1.0$
$\text{IG}=0.940-(\frac{8}{14}\times0.811+\frac{6}{14}\times1.0)=0.940-0.893=\mathbf{0.047}$ bits

K-Means & K-Means++

W6▼
1
Choose K random centroids $\mu_1\ldots\mu_K$
2
Assign: $c_i=\arg\min_k\|\mathbf{x}_i-\mu_k\|^2$
3
Update: $\mu_k=\frac{1}{|C_k|}\sum_{i\in C_k}\mathbf{x}_i$
4
Repeat until centroids stop moving
Objective (Inertia)
$$J=\sum_{k=1}^K\sum_{\mathbf{x}_i\in C_k}\|\mathbf{x}_i-\mu_k\|^2$$
K-Means++
Choose next centroid $\propto d(\mathbf{x})^2$ — favours far points, avoids bad local optima. Default in sklearn.

6 Limitations (all exam-testable):

  • Must choose K upfront
  • Assumes spherical, equal-size clusters
  • Sensitive to initialisation → K-Means++
  • Sensitive to feature scale → standardise first
  • Converges to local optimum (non-convex J)
  • Fails in high dimensions (curse of dimensionality)
Exam Trap
More K always lowers inertia — even K=N gives inertia=0. Inertia alone cannot choose K. Use the Elbow method (heuristic, not law).
Exam Q (Assignment 2 mirror)
Concentric rings: K-Means fails because it minimises Euclidean distance (assumes spherical clusters). Inner ring and outer ring have similar centroids → inseparable by K-Means.

Hierarchical Clustering (Agglomerative)

W7▼

Algorithm:

1
Each point = its own cluster
2
Merge the two closest clusters
3
Repeat until 1 cluster remains
4
Result: dendrogram tree — cut at height to get K clusters
Advantage
No need to specify K upfront. Cut dendrogram at any height to get desired K.
LinkageDistance Between ClustersShape Preference
Single-linkMin distance between any pair of pointsElongated, chained (rings ✅)
Complete-linkMax distance between any pair of pointsCompact, spherical
Average-linkMean of all pairwise distancesBalanced
Assignment 2 Q2 Type

For concentric rings: use single-linkage. Complete-linkage uses the maximum inter-cluster distance — this causes it to see the two rings as "far apart" because the max distance between outer ring and inner ring is large, forcing them to stay separate until very late, producing chain-like false merges.

Merge {18,22,25,42,27,43} using single-linkage. What are the first two merges?

Show Answer
Distances between all pairs → closest: 42 and 43 (d=1) → merge to {42,43}.
Next closest: 27 and 25 (d=2) → merge to {25,27}.
Then {18,22} (d=4), then {25,27} and {42,43} join via single-link distance, etc.

Anomaly Detection

W8▼

Isolation Forest:

  • Anomalies are isolated early → short path length in random trees
  • Normal points buried deep → long path length
Anomaly Score
$$s(x,n)=2^{-\frac{E[h(x)]}{c(n)}}$$

$c(n)=2H(n-1)-\frac{2(n-1)}{n}$, $H$=harmonic number

  • $s\approx1$ → anomaly
  • $s\approx0.5$ → normal
  • $s\approx0$ → very normal (buried deep)

Gaussian Model:

Density Estimation
$$p(\mathbf{x})=\prod_{j=1}^n p(x_j;\mu_j,\sigma_j^2)$$

Flag anomaly if $p(\mathbf{x})<\epsilon$

Gaussian PDF
$$p(x;\mu,\sigma^2)=\frac{1}{\sqrt{2\pi\sigma^2}}\exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$
Exam Trap
Use anomaly detection (not supervised learning) when positive examples are very rare (0–20 common) and future anomalies may look different from past ones.
