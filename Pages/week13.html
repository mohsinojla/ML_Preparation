Containers & Model Drift

Package your ML app with Docker for reproducibility, then learn to detect when real-world data distribution shifts away from your training distribution — causing model degradation.

🐍

Virtual Environments

▼

Virtual environments isolate Python dependencies per project, preventing version conflicts between projects.

Featurevenvconda
TypePython-onlyPython + system libraries (CUDA, etc.)
WeightLightweightHeavier
Package managerpipconda + pip
GPU/CUDA supportManual setup requiredNative: conda install cudatoolkit
Industry standard in DS/MLGeneral Python projectsYes — preferred for ML
Create commandpython -m venv envconda create -n myenv python=3.10
✅ Use conda for ML
Conda is the industry standard in data science and ML because it manages CUDA, MKL, and other system dependencies that pip cannot handle.
🐳

Docker Containers

▼

Docker packages your entire application — code, dependencies, system libraries, and runtime — into a container. Like a shipping container: the contents don't change regardless of what ship carries it.

Image
Read-only blueprint. Like a class. Built from Dockerfile. Contains everything needed to run the app.
Container
Running instance of an image. Like an object instantiated from a class. Isolated from the host.
Registry
Repository for images (Docker Hub, AWS ECR, GCP Artifact Registry). Pull to deploy anywhere.
✅ "Works on my machine" problem solved
With Docker, the container runs identically on dev laptop, CI server, and production cloud. The environment is part of the artifact.
📄

Dockerfile for ML APIs

▼
# Base Python image (slim = smaller footprint)
FROM python:3.10-slim

# Set working directory inside container
WORKDIR /app

# Copy requirements first (layer caching optimization)
COPY requirements.txt .
RUN pip install -r requirements.txt

# Copy application code
COPY . .

# Expose port for the API
EXPOSE 8000

# Start command
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
InstructionPurpose
FROMBase image to build upon. Provides OS + Python.
WORKDIRSets working directory for subsequent commands.
COPYCopy files from host into container filesystem.
RUNExecute a command during image build (install deps).
EXPOSEDeclare which port the container listens on.
CMDDefault command to run when container starts.

Data Drift (Covariate Shift)

▼

Data drift (covariate shift) occurs when the distribution of input features $P(X)$ changes between training and production, while the relationship $P(Y|X)$ stays the same.

Covariate shift: $P_{\text{train}}(X) \neq P_{\text{prod}}(X)$ but $P(Y|X)$ remains the same.
Example 1
User age distribution skews younger over time. Model trained on older user data may not serve young users as well.
Example 2
Model trained on US data, deployed globally. Feature distributions (income, spending patterns) differ across countries.
⚠ Even with the same P(Y|X), data drift causes performance degradation
The model was optimized for the training distribution. In underrepresented regions of the new distribution, it may perform poorly.
🔀

Concept Drift

▼

Concept drift is more severe: the relationship $P(Y|X)$ itself changes. What was a valid prediction is no longer correct for the same inputs.

Concept drift: $P_{\text{train}}(Y|X) \neq P_{\text{prod}}(Y|X)$
TypeDescriptionExample
SuddenAbrupt change, immediateCOVID-19 completely disrupts demand forecasting models overnight
GradualSlow shift over months/yearsCustomer tastes in music/fashion evolve, recommendation model becomes stale
IncrementalSteady progressive driftInflation erodes price thresholds in fraud detection (₹5000 was suspicious, now ₹50000 is normal)
RecurringPeriodic, cyclical patternHoliday shopping — spending patterns during December are very different from rest of year
🔑 Detection strategy
Monitor prediction distribution over time. Use statistical tests (KS test, Chi-square) to detect when feature distributions shift significantly from training distribution.
📊

Kolmogorov-Smirnov (KS) Test

▼

The KS test measures the maximum difference between two empirical cumulative distribution functions (ECDFs). Used to detect data drift between training and production distributions.

$$D = \sup_x |F_{\text{train}}(x) - F_{\text{prod}}(x)|$$ p-value < 0.05 → statistically significant drift detected.

This week introduces the concept; Week 14 goes deep with thresholds, the full example, and alternative tests.

Quiz — Containers & Drift

▼

Q1. In Docker, what is the difference between an image and a container?

Image : Container = Class : Object. The image is the blueprint (built from Dockerfile), the container is the live running instance.

Q2. Data drift (covariate shift) means:

Covariate shift: the input feature distribution P(X) changes (e.g., user demographics shift), but the true relationship P(Y|X) is unchanged.

Q3. COVID-19 suddenly disrupting demand forecasting models is an example of:

Sudden drift: an abrupt, immediate change in P(Y|X). COVID lockdowns changed consumer behavior overnight — a textbook example of sudden concept drift.

Q4. Which is the industry-standard virtual environment tool for ML/Data Science?

Conda is the industry standard in DS/ML. It manages Python AND system libraries like CUDA, MKL, etc. — critical for GPU deep learning. venv can only manage Python packages.

Q5. The Dockerfile instruction that runs during image build (to install packages) is:

RUN executes a command during build: RUN pip install -r requirements.txt. CMD is the default command when the container starts (at runtime, not build time).

Q6. In the KS test for drift detection, a p-value < 0.05 means:

p-value < 0.05 → we reject the null hypothesis (that the distributions are the same) → statistically significant drift is detected.

Numerical Problems

▼
Problem 1 — Identifying Drift Types
Classify each scenario as: data drift, sudden concept drift, gradual concept drift, incremental concept drift, or recurring concept drift. (a) Holiday shopping spike every December. (b) A fraud threshold of $500 was correct in 2018 but $2000 is now the typical transaction due to inflation. (c) User age distribution shifts from 35-50 to 18-25 over 3 years. (d) A global pandemic changes buying behavior overnight.
(a)Recurring concept drift — predictable seasonal pattern. P(Y|X) changes each December and reverts.
(b)Incremental concept drift — steady progressive drift. Inflation slowly erodes what constitutes a suspicious transaction amount.
(c)Data drift (covariate shift) — P(X) changes (younger users), but the actual relationship P(Y|X) may be the same.
(d)Sudden concept drift — abrupt change. COVID lockdowns changed P(Y|X) for demand models overnight.
Problem 2 — Docker Build & Run
You have a FastAPI ML app in /app/main.py with model.pkl. Write the commands to: (1) build the Docker image, (2) run it mapping host port 9000 to container port 8000, (3) verify it's running, (4) stop it.

Rundocker run -d -p 9000:8000 --name fraud-api ml-fraud-api  (-d = detached background mode)
Verifydocker ps  → see container ID and status. Then: curl http://localhost:9000/docs

Problem 3 — KS Test by Hand (Intro)
Training data: {2, 4, 6, 8}. Production data: {4, 6, 8, 10}. By inspection, does this look like drift? Which test would you apply and why?
VisualTraining ranges [2,8], production ranges [4,10]. The production distribution has shifted right by ~2 units — this looks like data drift.
Test to useKS test — because this is a continuous feature and we want to compare two empirical distributions.
KS statisticD = max|F_train(x) - F_prod(x)|. At x=4: F_train(4)=0.25, F_prod(4)=0.25, diff=0. At x=6: F_train=0.5, F_prod=0.5. At x=2: F_train=0.25, F_prod=0.0, diff=0.25. D ≈ 0.25. Week 14 covers the full KS calculation procedure.
