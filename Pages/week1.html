Introduction to ML, KNN & Gradient Descent

Build the mathematical foundation — understand what ML is, master K-Nearest Neighbours, learn feature scaling, and grasp gradient descent which powers almost every ML model.

🤖

What is Machine Learning?

▼

Arthur Samuel (1959): Machine Learning is the field of study that gives computers the ability to learn without being explicitly programmed.

Tom Mitchell (1997): A computer program is said to learn from experience E with respect to task T and performance measure P, if its performance at T, as measured by P, improves with E.

🔑 Core Idea
Traditional programming: Data + Rules → Output. Machine Learning: Data + Output → Rules (the model learns the rules itself).

Types of Machine Learning

📚 Supervised Learning

Learn from labelled data. Input-output pairs are given. Model learns to map inputs → outputs.

Examples: Regression, Classification, SVM

🔍 Unsupervised Learning

Learn from unlabelled data. Find hidden patterns or structure without guidance.

Examples: K-Means, PCA, Autoencoders

🎮 Reinforcement Learning

Agent learns by trial and error — takes actions in environment, receives rewards or penalties.

Examples: AlphaGo, Game-playing AI

Classification vs Regression

Classification: Output is a discrete class label (cat/dog, spam/not spam)

Regression: Output is a continuous number (house price, temperature)

Why does ML work?

Real-world data has patterns. ML algorithms find these patterns by optimizing a mathematical objective function using training data.

📍

K-Nearest Neighbours (KNN)

▼

KNN is a non-parametric, lazy learning algorithm. It makes predictions by finding the K most similar training examples (neighbours) and using their labels to classify or regress a new point.

It is "lazy" because it doesn't learn a model — it memorises the entire training set and only computes at prediction time.

💡 Core Assumption
Similar inputs tend to have similar outputs. "Birds of a feather flock together."

Distance Metrics

KNN needs to measure "nearness" between points. The choice of metric affects the results significantly.

Formula
Euclidean Distance (most common)
$$d(p, q) = \sqrt{\sum_{i=1}^{n}(p_i - q_i)^2}$$

In 2D: $d = \sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}$

Formula
Manhattan Distance (L1 norm)
$$d(p, q) = \sum_{i=1}^{n}|p_i - q_i|$$

Sum of absolute differences. Like navigating a city grid.

Formula
Minkowski Distance (general form)
$$d(p, q) = \left(\sum_{i=1}^{n}|p_i - q_i|^r\right)^{1/r}$$

$r=1$ → Manhattan, $r=2$ → Euclidean, $r=\infty$ → Chebyshev

MetricFormulaWhen to UseSensitivity to Outliers
Euclidean $\sqrt{\sum(p_i-q_i)^2}$ Continuous features, real-valued data High (squares differences)
Manhattan $\sum|p_i-q_i|$ High-dimensional data, sparse features Lower
Minkowski $(\sum|p_i-q_i|^r)^{1/r}$ General case, tune $r$ Depends on $r$

KNN Algorithm Steps

1
Choose K Select the number of neighbours K (hyperparameter). Odd K avoids ties in binary classification.
2
Calculate Distances Compute the distance from the new point to every training point using your chosen metric.
3
Find K Nearest Sort all distances and pick the K smallest — these are the K nearest neighbours.
4
Predict Classification: take the majority vote of K neighbours' labels. Regression: take the average of K neighbours' values.
📊 Choosing K

Small K (e.g. K=1): Very flexible, follows training data closely — risk of overfitting. Decision boundary is jagged.

Large K: Smoother decision boundary, but may underfit and miss local patterns. Computationally expensive.

Rule of thumb: $K \approx \sqrt{N}$ where $N$ is the number of training samples. Always use cross-validation to tune K.

⚠️ KNN Weakness
Slow at prediction time — must compute distance to all training points. Memory-intensive. Very sensitive to feature scale (always scale before KNN!).

Feature Scaling

▼

If Feature A = Income ($100,000) and Feature B = Age (25), Euclidean distance is dominated by Income even if Age is equally important. Feature scaling brings all features to a comparable range.

⚠️ Always Scale Before KNN
Without scaling, features with large magnitudes dominate distance calculations and the model ignores smaller-scale features entirely.
Formula
Min-Max Normalization
$$x' = \frac{x - x_{min}}{x_{max} - x_{min}}$$
Result
Scales all values to [0, 1]. Preserves the shape of the original distribution.

When to use: When you know the min/max bounds and have no outliers. Common for image pixel values (0–255 → 0–1).

Weakness: Sensitive to outliers. One extreme value stretches all others towards 0.

Formula
Z-Score Standardization (Standard Scaler)
$$x' = \frac{x - \mu}{\sigma}$$

where $\mu$ = mean, $\sigma$ = standard deviation

Result
Output has mean = 0 and standard deviation = 1. Called a "standard normal" distribution.

When to use: When data follows a Gaussian distribution or when outliers are present. Preferred for gradient descent algorithms.

PropertyMin-Max NormalizationZ-Score Standardization
Output range[0, 1](-∞, +∞), mean=0
Outlier sensitiveYes — heavilyLess sensitive
Preserves zerosNoNo

Best forNeural nets, image dataRegression, SVM, PCA

Gradient Descent

▼

Analogy: You're blindfolded on a hilly landscape. Your goal is to reach the lowest valley. At each step, you feel the ground beneath you and take a step in the direction that goes downhill. Gradient Descent is exactly this — an iterative optimisation algorithm that minimises a function by following the negative gradient.

🔑 Why Gradient Descent?
Most ML loss functions don't have a simple closed-form minimum. Gradient Descent is a general method that works for almost any differentiable function.

The Gradient

For a function $f(\theta)$, the gradient $\nabla f$ tells us the direction of steepest ascent. We move in the opposite direction (steepest descent) to minimise $f$.

Core Update Rule
Gradient Descent Parameter Update
$$\theta_{new} = \theta_{old} - \alpha \cdot \frac{\partial J(\theta)}{\partial \theta}$$

$\theta$ (theta)

The parameter(s) we are optimising — weights of the model.

$\alpha$ (alpha)

Learning rate — how big each step is. Critical hyperparameter.

$\frac{\partial J}{\partial \theta}$

Partial derivative of cost function w.r.t. $\theta$ — the gradient.

Choosing the Learning Rate $\alpha$

🐌 Too Small
Gradient descent converges very slowly. May take millions of iterations. Wastes computation.
💥 Too Large
Takes huge steps and overshoots the minimum. May diverge and the cost increases instead!
✅ Just Right
Converges in a reasonable number of steps. Common starting values: 0.1, 0.01, 0.001. Use a learning rate schedule to decrease $\alpha$ over time.

Types of Gradient Descent

TypeUpdate usingSpeedNoiseUse case
Batch GD All training samples Slow per update None (stable) Small datasets
Stochastic GD (SGD) 1 sample at a time Fast per update High (noisy) Online learning
Mini-Batch GD Batch of m samples Balanced Medium Deep learning (default)

Multivariable Gradient Descent

In real ML, we have multiple parameters $\theta_0, \theta_1, \ldots, \theta_n$. Gradient descent updates ALL parameters simultaneously at each step.

Multivariable Update
Simultaneous update for all parameters
$$\theta_j := \theta_j - \alpha \frac{\partial J(\boldsymbol{\theta})}{\partial \theta_j} \quad \text{for all } j$$
⚠️ Simultaneous Update
All $\theta_j$ must be updated at the same time using the old values. Do NOT update $\theta_0$ and then use the new $\theta_0$ to update $\theta_1$.
🔑 Gradient Vector
The gradient of a multivariable function is a vector: $\nabla J = \left[\frac{\partial J}{\partial \theta_0}, \frac{\partial J}{\partial \theta_1}, \ldots, \frac{\partial J}{\partial \theta_n}\right]$. GD moves in the direction of $-\nabla J$.

Week 1 Quiz

▼

🧩 Test Your Understanding

Q1. What does KNN predict for a new point with K=3, and neighbours have labels [A, A, B]?

KNN uses majority voting. 2 neighbours are class A, 1 is class B, so the prediction is A.

Q2. Min-Max normalization scales features to which range?

Min-Max normalization uses the formula $(x - x_{min})/(x_{max} - x_{min})$, which always produces values in [0, 1].

Q3. What happens if the learning rate in gradient descent is too large?

A too-large learning rate causes gradient descent to overshoot the minimum, potentially causing the cost to increase indefinitely (divergence).

Q4. Euclidean distance between points (1, 2) and (4, 6) is:

$d = \sqrt{(4-1)^2 + (6-2)^2} = \sqrt{9 + 16} = \sqrt{25} = 5$

Q5. KNN is called a "lazy" learner because:

Lazy learners don't build an explicit model during training. Instead they store all training data and do computation only when a prediction is requested.

Problem 1 — KNN Classification

Exam style

Given training data:

PointFeature 1 (x₁)Feature 2 (x₂)Label
A11Class 0
B22Class 0
C34Class 1
D55Class 1
E42Class 1

Classify the new point Q = (3, 3) using K=3 and Euclidean distance.

Step 1
Calculate Euclidean distance from Q=(3,3) to each point:
$d(Q, A) = \sqrt{(3-1)^2 + (3-1)^2} = \sqrt{4+4} = \sqrt{8} \approx 2.83$
$d(Q, B) = \sqrt{(3-2)^2 + (3-2)^2} = \sqrt{1+1} = \sqrt{2} \approx 1.41$
$d(Q, C) = \sqrt{(3-3)^2 + (3-4)^2} = \sqrt{0+1} = 1.00$
$d(Q, D) = \sqrt{(3-5)^2 + (3-5)^2} = \sqrt{4+4} = \sqrt{8} \approx 2.83$
$d(Q, E) = \sqrt{(3-4)^2 + (3-2)^2} = \sqrt{1+1} = \sqrt{2} \approx 1.41$
Step 2
Sort by distance and pick K=3 nearest:
1st: C (d=1.00) → Class 1
2nd: B (d=1.41) → Class 0
3rd: E (d=1.41) → Class 1
(A and D are 4th and 5th with d≈2.83)
Step 3
Majority vote among 3 neighbours: {C=Class 1, B=Class 0, E=Class 1}
Class 1: 2 votes | Class 0: 1 vote
✅ Prediction: Class 1

Problem 2 — Feature Scaling

Apply Min-Max normalization to the following feature values:
Data = [10, 20, 30, 40, 50]. Normalize each value.
Step 1
Find min and max: $x_{min} = 10$, $x_{max} = 50$
Step 2
Apply formula $x' = \frac{x - 10}{50 - 10} = \frac{x - 10}{40}$:
$10 \to \frac{10-10}{40} = 0.00$
$20 \to \frac{20-10}{40} = 0.25$
$30 \to \frac{30-10}{40} = 0.50$
$40 \to \frac{40-10}{40} = 0.75$
$50 \to \frac{50-10}{40} = 1.00$
Result
Normalized: [0.0, 0.25, 0.5, 0.75, 1.0] ✅

Problem 3 — Gradient Descent Iteration

Given cost function $J(\theta) = \theta^2$, starting at $\theta_0 = 4$, learning rate $\alpha = 0.3$.
Perform 3 iterations of gradient descent and find the final $\theta$.
Setup
$J(\theta) = \theta^2$, so $\frac{dJ}{d\theta} = 2\theta$
Update rule: $\theta := \theta - \alpha \cdot 2\theta = \theta(1 - 2\alpha) = \theta(1 - 0.6) = 0.4\theta$
Iter 1
$\theta_1 = \theta_0 - 0.3 \times 2\theta_0 = 4 - 0.3(8) = 4 - 2.4 = \mathbf{1.6}$ | $J = 1.6^2 = 2.56$
Iter 2
$\theta_2 = 1.6 - 0.3 \times 2(1.6) = 1.6 - 0.96 = \mathbf{0.64}$ | $J = 0.64^2 = 0.41$
Iter 3
$\theta_3 = 0.64 - 0.3 \times 2(0.64) = 0.64 - 0.384 = \mathbf{0.256}$ | $J = 0.0655$
Result
After 3 iterations: $\theta \approx 0.256$, approaching the minimum at $\theta = 0$. ✅
