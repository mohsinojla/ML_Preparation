Decision Trees

Shift from gradient descent to combinatorial optimisation. Decision trees minimise impurity — no gradients, no convexity — just recursive splitting that any human can read and interpret.

🔀

From Continuous to Discrete Optimization

▼

So far (Weeks 1–4): We used Gradient Descent — minimise a smooth, differentiable loss over continuous parameters (weights, $\theta$, $\mathbf{w}$).

Decision Trees are completely different: No gradients. No convexity. We optimise a combinatorial structure — which feature to split on, and where to split — by minimising impurity.

Gradient Descent

Continuous parameters, smooth loss, $\theta \leftarrow \theta - \alpha\nabla J$

Decision Trees

Discrete splits, no gradients — minimise impurity greedily at each node

What we optimise

Choose best feature + best split point to maximise information gain

🔑 Key Idea
Decision Trees build a model that is a tree of if-else rules. They are greedy — at each node, they pick the locally best split. The resulting model is fully human-interpretable.
🌳

Decision Tree Structure

▼

Root Node

Top node — the first feature split. Covers the entire dataset.

Internal Nodes

Each tests one feature. Data flows left or right based on the test result.

Leaf Nodes

Terminal nodes — hold the class prediction (majority class or probability).

Categorical Features

When a feature is categorical (e.g., Outlook = {Sunny, Overcast, Rain}), the tree branches for each distinct value. Each branch leads to a subtree or leaf.

Example: Play Tennis
Root split on Outlook: Overcast → always Yes (pure leaf). Sunny → check Humidity. Rain → check Wind.

Continuous Features

For continuous features (e.g., Temperature), the tree finds a threshold $t$ and splits: $x \leq t$ → left branch, $x > t$ → right branch. The best threshold is chosen to maximise information gain.

Split Condition
Binary split on continuous feature
$$\text{Left branch: } x_j \leq t \qquad \text{Right branch: } x_j > t$$

Best $t$ is typically the midpoint between adjacent unique values. Try all $n-1$ splits, pick the best IG.

Decision Boundary (From Lecture)

Directly from lecture: "Decision trees divide the feature space into axis-parallel (hyper-)rectangles. Each rectangular region is labelled with one class label or a probability distribution over labels."

Decision Tree → Axis-Parallel Rectangular Decision Regions

Each split creates a horizontal or vertical cut — unlike SVM which can cut at any angle

📌 DT vs SVM Decision Boundary
SVM: diagonal hyperplane (any angle). Decision Tree: always axis-parallel — splits are always "feature $j \leq t$" not a linear combination of features.

Expressiveness

From lecture: Decision trees can represent any boolean function of the input features. In the worst case, the tree requires exponentially many nodes.

✅ Very Expressive

Any boolean function can be represented. A depth-$d$ tree can represent $2^{2^d}$ different functions.

⚠️ Complexity Grows Fast

Worst case: need exponentially many nodes. This is why pruning is essential to avoid overfitting.

Feature Selection — Choosing the Best Split

▼

Key Problem (from lecture): At each node, which feature do we split on? This is the core algorithmic decision. Several strategies exist:

StrategyDescriptionUsed By
RandomSelect any feature at randomBaseline, Random Forests
Least-ValuesFeature with fewest distinct values (e.g., binary features)Rare
Most-ValuesFeature with most distinct valuesRare (often overfits)
Max-GainFeature with largest expected information gainID3 ← exam focus
🔑 ID3 Uses Max-Gain
The ID3 algorithm (Quinlan, 1986) uses Information Gain to select the best feature at each split. C4.5 improves on ID3 by using Gain Ratio to correct bias towards high-cardinality features.

Entropy — Measuring Impurity

Entropy measures the impurity or uncertainty in a set of examples. A pure node (all one class) has entropy 0. A perfectly mixed node (50/50 split) has maximum entropy.

Entropy Formula
Shannon Entropy (from lecture — K classes)
$$H(S) = -\sum_{k=1}^{K} p_k \log_2 p_k$$

$p_k$ = fraction of examples in class $k$. Convention: $0 \cdot \log_2 0 = 0$. Units: bits.

Pure node: $H = 0$

All examples same class: $p_1=1$, all others $=0$.
$H = -(1\log_2 1) = 0$

Maximally mixed: $H = 1$

Equal split (binary): $p_+ = p_- = 0.5$.
$H = -(0.5\log_2 0.5 \times 2) = 1$ bit

Entropy vs class probability $p$ (binary case)

Maximum entropy = 1 bit at $p=0.5$ | Zero entropy at $p=0$ or $p=1$

Information Gain — The ID3 Criterion

Information Gain measures how much a feature reduces entropy. We want the feature that reduces uncertainty the most.

Information Gain
IG of feature A on dataset S
$$\text{IG}(S, A) = H(S) - \sum_{v \in \text{Values}(A)} \frac{|S_v|}{|S|} H(S_v)$$

$S_v$ = subset of $S$ where feature $A$ takes value $v$ | $\frac{|S_v|}{|S|}$ = weight of each branch

1
Compute $H(S)$ — entropy of the full dataset before the split.
2
For each feature $A$, compute weighted entropy after splitting on $A$: $\sum_v \frac{|S_v|}{|S|} H(S_v)$
3
IG = before − after. Pick the feature with highest IG.
4
Recurse on each branch with the remaining features until leaves are pure or stopping criterion met.
⚠️ IG Bias Problem
Information Gain is biased towards features with many distinct values. A feature like "ID" (unique per example) gives perfect IG but is useless for generalisation. Solution: use Gain Ratio.

Gain Ratio (C4.5 Improvement)

Gain Ratio normalises Information Gain by the Split Information — the intrinsic information of the feature itself. Features with many values have high Split Info, which penalises them.

Split Information
Intrinsic info of feature A
$$\text{SplitInfo}(S, A) = -\sum_{v \in \text{Values}(A)} \frac{|S_v|}{|S|} \log_2 \frac{|S_v|}{|S|}$$
Gain Ratio
IG normalised by SplitInfo
$$\text{GainRatio}(S, A) = \frac{\text{IG}(S, A)}{\text{SplitInfo}(S, A)}$$

From lecture: "If a feature has many possible values, the Split Information value will be high, which shrinks the final Gain Ratio."

CriterionFormulaBiasAlgorithm
Info Gain$H(S) - \sum_v \frac{|S_v|}{|S|}H(S_v)$Favours many-valued featuresID3
Gain Ratio$\text{IG} / \text{SplitInfo}$Corrects IG biasC4.5
Gini Index$1 - \sum_k p_k^2$Similar to IG but no logCART
Gini Index
Alternative impurity measure used by CART / sklearn
$$\text{Gini}(S) = 1 - \sum_{k=1}^{K} p_k^2$$

Gini is computationally cheaper (no $\log$). Used by sklearn's DecisionTreeClassifier by default.

Overfitting in Decision Trees

▼

Decision Trees are notorious overfitters. An unrestricted tree will grow until every leaf is pure — perfectly fitting training data (including noise) but failing on test data.

⚠️ The Extreme Case
From lecture: "it is always possible to fit a polynomial model of order $n-1$ perfectly to $n$ data points" — equivalently, a full decision tree can achieve 0 training error by splitting until each leaf has exactly 1 example. This is pure memorisation.

Signs of DT Overfitting

  • Very deep tree
  • Many leaves with 1 example each
  • High training accuracy, low test accuracy
  • Splits on noisy/irrelevant features

Solutions

  • Pre-pruning (early stopping)
  • Post-pruning (grow then trim)
  • Ensemble methods (Random Forest)
  • Minimum samples per split/leaf

Early Stopping Criteria (From Lecture)

CriterionStop when…Effect
max_depthTree reaches depth $d$Hard limit on complexity
min_samples_splitNode has fewer than $m$ examplesPrevents tiny splits
min_samples_leafAny leaf would have fewer than $l$ examplesEnsures meaningful leaves
IG < thresholdBest split gain below $\epsilon$Stops uninformative splits
📌 Lecture Interpretation
"Hard constraints on hypothesis space" — pre-pruning restricts what the tree can learn upfront.
✂️

Pruning Strategies

▼

From lecture: "Pre-pruning (early stopping) halts tree growth using constraints… it's fast and memory-efficient but can underfit by cutting useful splits. Post-pruning grows a full tree then removes branches using validation-based methods — usually yields better generalisation but is slower."

From lecture: "Is basically early stopping… 'Stop learning now, because if you keep going, you're just going to start memorizing the noise in the background.'"

✅ Advantages
Fast — stops early instead of building a big tree. Memory-efficient. Simple to implement.
⚠️ Risk: Short-Sightedness
From lecture: "Pre-pruning is 'short-sighted.' It might stop a split because the immediate improvement is low — even if that split would have enabled excellent deeper splits."
1
Set max_depth = d — tree cannot grow deeper than $d$ levels from root.
2
Set min_samples_split — only split a node if it has ≥ $m$ examples.
3
Check IG threshold — only split if IG ≥ $\epsilon$. Ignores very small gains.
4
Return leaf when any stopping criterion is met — predict majority class.

From lecture: "Post-pruning grows a full tree then removes branches… 'hindsight' — it builds the whole tree, then looks back and says 'this specific branch led nowhere; let's cut it.'"

✅ Better Generalisation
Sees the whole tree first, then makes informed trim decisions — usually yields better test accuracy than pre-pruning.
⚠️ Risk: Expensive
Must build the full (possibly huge) tree first, then evaluate pruning candidates. Needs a held-out validation set or cross-validation.

Mathematics of Post-Pruning (Cost-Complexity)

Cost-Complexity Criterion
Objective to minimise for optimal pruning
$$R_\alpha(T) = R(T) + \alpha |T|$$

$R(T)$ = misclassification rate on training/validation set | $|T|$ = number of leaf nodes | $\alpha$ = complexity penalty

🔑 Intuition
We want the simplest tree (small $|T|$) that still fits the data well (small $R(T)$). $\alpha$ controls this tradeoff — like regularization $\lambda$ in regression. Larger $\alpha$ → more pruning → simpler tree.

Which to Use? (From Lecture)

✅ Modern Practice
From lecture: "In modern practice (like XGBoost or LightGBM), we often use a bit of both. We use pre-pruning (max depth) to keep the model from exploding in size, and then use post-pruning (regularization/gamma) to fine-tune the complexity."

Q1. Entropy of a node with 6 positive and 6 negative examples (binary classification) is:

With equal class proportions ($p=0.5$), entropy is maximised: $H = -(0.5\log_2 0.5 + 0.5\log_2 0.5) = -(-0.5 - 0.5) = 1$ bit.

Q2. The ID3 algorithm selects features using:

From the lecture: "The ID3 algorithm uses the Max-Gain method of selecting the best features." C4.5 improved on ID3 by using Gain Ratio instead.

Q3. Why does Information Gain have a bias problem?

A feature like "ID" (unique per example) gives perfect IG but is useless generalization-wise. Gain Ratio corrects this by dividing by the SplitInfo of the feature.

Q4. Decision tree decision boundaries are always:

From the lecture: "Decision trees divide the feature space into axis-parallel (hyper-)rectangles." Each split is always "feature $j \leq t$" — never a linear combination of features.

Q5. Pre-pruning is described as "short-sighted" because:

Pre-pruning makes greedy local decisions. A split with low immediate gain might enable excellent splits below it — but pre-pruning never explores that possibility.

Q6. A leaf node has 0 examples from class A and 5 from class B. Its entropy is:

$H = -(0\cdot\log_2 0 + 1\cdot\log_2 1) = 0$ bits. A pure node (all one class) has zero entropy — no uncertainty.

Numerical Problems

▼

Problem 1 — Compute Entropy

A dataset has 9 positive examples (+) and 5 negative examples (−). Compute the entropy $H(S)$.
Step 1
Total examples: $|S| = 9 + 5 = 14$
$p_+ = \frac{9}{14} \approx 0.643$    $p_- = \frac{5}{14} \approx 0.357$
Step 2
$H(S) = -p_+ \log_2 p_+ - p_- \log_2 p_-$
$= -(0.643)\log_2(0.643) - (0.357)\log_2(0.357)$
$= -(0.643)(-0.637) - (0.357)(-1.486)$
$= 0.410 + 0.530$
$= \mathbf{0.940 \text{ bits}}$

Problem 2 — Information Gain Calculation

Dataset $S$ has 14 examples: 9 positive, 5 negative ($H(S) = 0.940$ bits).
Feature Wind splits $S$ into:
— $S_{\text{weak}}$: 6 positive, 2 negative (8 total)
— $S_{\text{strong}}$: 3 positive, 3 negative (6 total)
Compute $\text{IG}(S, \text{Wind})$.
Step 1
$H(S_{\text{weak}}) = -\frac{6}{8}\log_2\frac{6}{8} - \frac{2}{8}\log_2\frac{2}{8}$
$= -0.75(-0.415) - 0.25(-2) = 0.311 + 0.500 = \mathbf{0.811}$ bits
Step 2
$H(S_{\text{strong}}) = -\frac{3}{6}\log_2\frac{3}{6} - \frac{3}{6}\log_2\frac{3}{6}$
$= -(0.5)(-1) - (0.5)(-1) = \mathbf{1.0}$ bits
Step 3
Weighted entropy after split:
$\frac{8}{14}(0.811) + \frac{6}{14}(1.0) = 0.464 + 0.429 = 0.893$ bits
Step 4
$\text{IG}(S, \text{Wind}) = H(S) - \text{weighted entropy} = 0.940 - 0.893 = \mathbf{0.047 \text{ bits}}$

Problem 3 — Gini Index

Compute the Gini Index for a node with class proportions: Class A = 40%, Class B = 35%, Class C = 25%.
Formula
$\text{Gini} = 1 - \sum_k p_k^2 = 1 - (p_A^2 + p_B^2 + p_C^2)$
Compute
$= 1 - (0.40^2 + 0.35^2 + 0.25^2)$
$= 1 - (0.16 + 0.1225 + 0.0625)$
$= 1 - 0.345 = \mathbf{0.655}$
Interpret
Gini = 0.655 → fairly impure node (high impurity). A pure node gives Gini = 0; equal split gives Gini = $1 - \frac{1}{K}$ for $K$ classes.

Problem 4 — Cost-Complexity Pruning

Tree $T_1$ has 10 leaves and error rate $R(T_1) = 0.08$.
Pruned tree $T_2$ has 4 leaves and error rate $R(T_2) = 0.12$.
With $\alpha = 0.01$, which tree is preferred?
$T_1$
$R_\alpha(T_1) = R(T_1) + \alpha|T_1| = 0.08 + 0.01 \times 10 = 0.08 + 0.10 = \mathbf{0.18}$
$T_2$
$R_\alpha(T_2) = R(T_2) + \alpha|T_2| = 0.12 + 0.01 \times 4 = 0.12 + 0.04 = \mathbf{0.16}$
Decision
$R_\alpha(T_2) = 0.16 < R_\alpha(T_1) = 0.18$ → ✅ Prefer the pruned tree $T_2$. Despite higher raw error, it is better when complexity is penalised.
