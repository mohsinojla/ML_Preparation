MLOps: Model Deployment & FastAPI

87% of ML models never reach production (Gartner). Bridge the gap between a trained model and a live production API using MLOps principles and FastAPI.

🚀

Why Deployment Matters

▼
📊 Gartner Statistic
87% of ML models never reach production. Most models die in Jupyter notebooks, never delivering business value. The gap between "model works in lab" and "model serves users" is where most ML projects fail.
✅ A model only creates value when it serves predictions
Building a 99% accurate model that nobody uses is worthless. Deployment = turning your model into a product.
Netflix
Recommendation models serve 200M+ users in real time.
Banking
Fraud detection models score every transaction in milliseconds.
Healthcare
X-ray AI assists radiologists — model must be embedded in clinical workflow.
⚙

What is MLOps?

▼

MLOps = ML + DevOps + Data Engineering. A set of practices to reliably deploy, monitor, and maintain ML models in production at scale.

ML Component
Model training, evaluation, versioning, experiment tracking.
DevOps Component
CI/CD pipelines, containerization, automated testing, deployment.
Data Engineering
Data pipelines, feature stores, data validation, monitoring.
🔄

MLOps Lifecycle — 7 Stages

▼
1
Data Collection — Gather raw data from databases, APIs, sensors, user interactions.
2
Data Preparation — Clean, transform, validate, and split data. Build reproducible pipelines.
3
Model Training — Train models with tracked experiments (MLflow), hyperparameter tuning, versioning.
4
Model Evaluation — Validate accuracy, fairness, robustness on holdout sets and business metrics.
5
Model Packaging — Serialize model (pickle/joblib), containerize with Docker, create API.
6
Deploy & Serve — Deploy to cloud/on-prem. Serve predictions via REST API, batch jobs, or edge devices.
7
Monitor & Retrain — Track performance, detect drift, trigger retraining when model degrades.
💾

Model Serialization

▼

After training, the model must be saved to disk so it can be loaded by the serving API without retraining. Two common options in Python:

LibraryUsageBest for
picklepickle.dump(model, f) / pickle.load(f)Quick and simple. Built into Python.
joblibjoblib.dump(model, 'model.pkl') / joblib.load('model.pkl')sklearn models — handles large numpy arrays more efficiently than pickle.
⚠ pickle security warning
Never unpickle files from untrusted sources. A malicious pickle file can execute arbitrary code on load. Use model cards and checksums to verify model files in production.

REST API Concepts

▼

A REST API allows any client (web, mobile, other services) to call your model over HTTP. Think of REST as the rules of the road and FastAPI as the car.

HTTP MethodPurposeExample
GETRetrieve data / readGET /model/info
POSTSend data, create, computePOST /predict
PUTUpdate existing resourcePUT /model/update
DELETERemove resourceDELETE /model/v1
📌 For ML inference
Use POST /predict: client sends features in request body (JSON), server returns prediction in response body (JSON).
⚡

FastAPI

▼

FastAPI is a modern, fast Python web framework for building APIs. It uses Python type hints for automatic request validation and generates interactive documentation (Swagger UI) automatically.

from fastapi import FastAPI
from pydantic import BaseModel
import joblib

app = FastAPI()
model = joblib.load("model.pkl")

class Features(BaseModel):
    data: list[float]

@app.post("/predict")
def predict(features: Features):
    pred = model.predict([features.data])
    return {"prediction": pred[0]}
FastAPI
The web framework. Handles routing, validation, serialization.
Pydantic
Data validation. Defines the request schema and validates incoming data types automatically.
Uvicorn
ASGI server. Runs the FastAPI app: uvicorn main:app --reload
✅ FastAPI advantages for ML
Automatic Swagger docs at /docs, async support for concurrent requests, built-in data validation, fast (comparable to Node.js), type-safe.

Quiz — MLOps & FastAPI

Q1. What percentage of ML models never reach production, according to the Gartner stat from the lecture?

87% of ML models never reach production (Gartner). This highlights the importance of MLOps — the gap between notebook and production is where most ML projects fail.

Q2. In the FastAPI code from the lecture, what does the Pydantic BaseModel do?

Pydantic BaseModel defines the expected input structure (Features with data: list[float]) and automatically validates incoming request data.

Q3. MLOps is the combination of:

MLOps = ML + DevOps + Data Engineering — combining model development, deployment automation, and data pipeline management.

Q4. Which HTTP method is used for prediction requests in a REST ML API?

POST is used to send data to the server. Prediction requests send feature data in the request body — POST /predict is the standard pattern.

Q5. Which library is recommended for saving sklearn models in the lecture?

joblib is recommended for sklearn models as it handles large numpy arrays more efficiently. The lecture shows: joblib.load("model.pkl") in the FastAPI example.

Q6. What is the 7th and final stage of the MLOps lifecycle?

The lifecycle doesn't end at deployment. Stage 7 is Monitor & Retrain — track performance in production, detect drift, and trigger retraining when the model degrades.

Problem 3 — API Throughput Calculation
A fraud detection API receives 1,000 transactions/second. Each prediction takes 10ms. How many parallel workers do you need? If each worker costs $0.05/hour, what is the hourly cost?
ThroughputEach worker can handle 1000ms/10ms = 100 requests/second.
Workers needed1000 req/s ÷ 100 req/s/worker = 10 workers minimum.
With headroomAdd 20% buffer: ceil(10 × 1.2) = 12 workers for traffic spikes.
Cost12 workers × $0.05/hr = $0.60/hr = $14.40/day = $432/month.
