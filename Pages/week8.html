Week 8

Anomaly Detection

Detect rare, unusual events in data — from aircraft engine faults to credit card fraud — using Isolation Forest and Gaussian density estimation.

🚨

What is Anomaly Detection?

▼

Anomaly detection (also called outlier detection) identifies data points that differ significantly from the majority. These unusual points may signal important events: fraud, equipment failure, network intrusion, or medical conditions.

Fraud Detection
Unusual transaction patterns on a credit card differ from a user's normal behavior.
Aircraft Engines
Abnormal heat or vibration readings indicate a developing engine fault.
Network Security
Unusual traffic spikes or access patterns signal a potential intrusion.
🔑 Core challenge
Anomalies are rare by definition. A dataset may have thousands of normal examples but only a handful of anomalies. Traditional classification fails because labels are scarce.
📐

Distance Metric Failure

▼

A naive approach: flag points that are "far" from other points using distance metrics. This fails in high dimensions due to the curse of dimensionality.

⚠ Relative Contrast Problem
In high-dimensional spaces, the ratio of maximum to minimum distance between points approaches 1. All points appear roughly equidistant — distance loses meaning as a discriminator.
Relative Contrast: $$\text{Relative Contrast} = \frac{d_{\max} - d_{\min}}{d_{\min}} \to 0 \text{ as dimensions } \to \infty$$

This is why Isolation Forest and Gaussian models are preferred over pure distance-based methods for high-dimensional anomaly detection.

🌲

Isolation Forest

▼

Isolation Forest exploits the fact that anomalies are few and different. An anomaly is easy to isolate — it takes very few splits in a random tree to separate it from all other points.

1
Build many random decision trees (iTree). At each node, randomly select a feature and a random split value.
2
For each data point, record the path length h(x) — how many splits it takes to isolate it.
3
Average path length E(h(x)) over all trees. Anomalies have short average paths; normal points have longer paths.
4
Compute anomaly score and compare to threshold 0.5.
Anomaly Score: $$s(x, n) = 2^{-\frac{E(h(x))}{c(n)}}$$ Normalisation factor: $$c(n) = 2H(n-1) - \frac{2(n-1)}{n}, \quad H(k) = \ln(k) + 0.5772$$ where $H$ is the harmonic number approximation.
Score rangeInterpretation
$s \approx 1$Definitely anomalous — very short isolation path
$s > 0.5$Likely anomalous
$s \approx 0.5$No clear anomaly signal
$s \approx 0$Definitely normal — long path needed to isolate
✅ Why Isolation Forest works
No distance computation needed. Works well in high dimensions. $O(n \log n)$ — fast and scalable. Handles mixed features.
📈

Gaussian Model for Anomaly Detection

▼

Model the probability density of normal examples. If a new point has very low probability under this model, flag it as an anomaly.

Gaussian probability density (per feature): $$p(x_j; \mu_j, \sigma_j^2) = \frac{1}{\sqrt{2\pi\sigma_j^2}} \exp\!\left(-\frac{(x_j - \mu_j)^2}{2\sigma_j^2}\right)$$ Product over all features (assuming independence): $$p(\mathbf{x}) = \prod_{j=1}^{n} p(x_j; \mu_j, \sigma_j^2)$$ Flag as anomaly if: $$p(\mathbf{x}) < \varepsilon$$
1
Compute $\mu_j = \frac{1}{m}\sum_{i=1}^m x_j^{(i)}$ — mean of each feature from training data.
2
Compute $\sigma_j^2 = \frac{1}{m}\sum_{i=1}^m (x_j^{(i)} - \mu_j)^2$ — variance of each feature.
3
For new point $\mathbf{x}$, compute $p(\mathbf{x}) = \prod_j p(x_j; \mu_j, \sigma_j^2)$.
4
If $p(\mathbf{x}) < \varepsilon$, flag as anomaly. Choose $\varepsilon$ on a cross-validation set.
Aircraft Engine Example
Features: x1 = heat generated, x2 = vibration intensity. Fit Gaussians to both. A new engine reading with unusual heat and vibration gets low $p(\mathbf{x})$ and is flagged.
📋

Algorithm Evaluation

▼

Since anomaly detection uses unlabeled training data, evaluation requires a labeled validation set with known anomalies. We then tune $\varepsilon$ on this set.

1
Train on normal examples only (unlabeled): fit $\mu_j, \sigma_j^2$.
2
Hold a cross-validation set with a few anomaly examples and normal examples.
3
Tune $\varepsilon$ to maximize F1-score on the cross-validation set (not accuracy — classes are imbalanced).
4
Evaluate final model on test set.
⚖

Anomaly Detection vs Supervised Learning

▼
CriterionUse Anomaly DetectionUse Supervised Learning
Positive examplesVery rare (0–20 total)Many (hundreds+)
Anomaly typesMany unknown types — future anomalies may look completely differentKnown types, future anomalies similar to training ones
Negative examplesMany normal examplesMany negative examples
Example use casesFraud, aircraft faults, network intrusionEmail spam, weather classification
⚠ Key insight
If your positive class (anomalies) is extremely rare and you expect future anomalies to look very different from past ones, anomaly detection is the right choice — not supervised classification.

Quiz — Anomaly Detection

▼

Q1. In Isolation Forest, anomalies have:

Anomalies are few and different — they are easy to isolate with few splits. Normal points blend in and take more splits to isolate.

Q2. An Isolation Forest score of s ≈ 1 means:

Score near 1 means very short isolation path — the point was easy to separate from all others, indicating an anomaly.

Q3. In the Gaussian anomaly model, you flag a point as an anomaly when:

Low probability density under the fitted Gaussian means the point is unusual. We flag it when p(x) falls below the threshold ε.

Q4. Anomaly detection is preferred over supervised learning when:

When anomalies are very rare and may look different in the future, anomaly detection is better than supervised learning which needs many labeled examples.

Q5. Why do distance metrics fail for anomaly detection in high dimensions?

In high dimensions, the ratio (d_max - d_min)/d_min approaches zero. All distances become similar, making it impossible to distinguish anomalies by distance alone.

Q6. To tune ε in the Gaussian anomaly model, you should:

Use F1-score (not accuracy) because classes are imbalanced. Tune ε on a held-out labeled validation set with a few known anomalies.

Numerical Problems

▼
Problem 1 — Gaussian Probability Density
Given a feature with mean μ=5 and variance σ²=2. Compute p(x=3). Is this an anomaly if ε=0.05?
Formula$p(x;\mu,\sigma^2) = \frac{1}{\sqrt{2\pi\sigma^2}}\exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$
Exponent$(x-\mu)^2 = (3-5)^2 = 4$. Exponent = $-4/(2 \times 2) = -1$. $e^{-1} \approx 0.368$
Normaliser$\sqrt{2\pi\sigma^2} = \sqrt{2\pi \times 2} = \sqrt{4\pi} \approx 3.545$
Result$p(x=3) = 0.368 / 3.545 \approx 0.1038$
Anomaly?$p = 0.1038 > \varepsilon = 0.05$. Not an anomaly — x=3 is within normal range.
Problem 2 — Isolation Forest Score
Given E(h(x))=3 (average path length) and n=100 (sample size). Compute the anomaly score s(x, n).
Step 1Compute $H(n-1) = H(99) \approx \ln(99) + 0.5772 \approx 4.595 + 0.5772 = 5.172$
Step 2$c(n) = 2H(n-1) - 2(n-1)/n = 2(5.172) - 2(99)/100 = 10.344 - 1.98 = 8.364$
Step 3$s(x,n) = 2^{-E(h(x))/c(n)} = 2^{-3/8.364} = 2^{-0.3587}$
Result$s = 2^{-0.3587} \approx 0.781$. Since s > 0.5, this point is flagged as anomalous.
Problem 3 — Multivariate Gaussian Detection
Aircraft engine with 2 features. Feature 1: μ₁=80°C, σ₁²=4. Feature 2: μ₂=0.5mm/s, σ₂²=0.01. New reading: x₁=75°C, x₂=0.6mm/s. Compute p(x) and check ε=0.001.
p(x₁)$(75-80)^2=25$. Exponent=$-25/8=-3.125$. $e^{-3.125}\approx 0.0439$. $\sqrt{2\pi\times4}\approx5.013$. $p(x_1)\approx0.00876$
p(x₂)$(0.6-0.5)^2=0.01$. Exponent=$-0.01/0.02=-0.5$. $e^{-0.5}\approx0.6065$. $\sqrt{2\pi\times0.01}\approx0.2507$. $p(x_2)\approx2.419$
p(x)$p(\mathbf{x}) = p(x_1)\times p(x_2) \approx 0.00876 \times 2.419 \approx 0.0212$
Anomaly?$p(\mathbf{x}) = 0.0212 > \varepsilon = 0.001$. Not anomalous — temperature is a bit low but within range.
