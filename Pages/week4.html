SVM — Soft Margin & Kernel Trick

Real data is messy. Extend SVM to handle non-separable data with slack variables, then unleash the power of kernels to classify non-linear data in high-dimensional spaces.

⚠️

Problems with Hard Margin SVM

▼

From the lecture: The hard margin classifier has a "complete lack of flexibility." Real-world data is rarely perfectly linearly separable — it has noise, outliers, and complex boundaries.

❌ Non-Linearly Separable

If classes overlap, no hyperplane can separate them without error — hard margin SVM has no solution.

❌ Outlier Sensitivity

A single outlier can drastically move the decision boundary or make the problem infeasible.

❌ Overfitting Risk

Trying to perfectly fit every training point creates a tight, fragile boundary — poor generalisation.

Hard Margin vs Soft Margin: Effect of Outliers

Left: Hard margin forced to adapt to outlier (small margin). Right: Soft margin ignores outlier (large margin).

💡 Solution: Allow Some Misclassification
Instead of requiring all points to be outside the margin, we allow some points to be inside the margin or even on the wrong side — but we penalise them. This gives the model flexibility while still controlling errors.
🌊

Soft Margin SVM

▼

Slack Variables $\xi_i$ (From Lecture)

We introduce a slack variable $\xi_i \geq 0$ for each training point. It measures how much the constraint is violated for point $i$.

Soft Constraint
Modified Constraint with Slack Variable
$$y_i(\mathbf{w}^T\mathbf{x}_i + b) \geq 1 - \xi_i \quad \forall i, \quad \xi_i \geq 0$$

$\xi_i = 0$

Point correctly classified and outside the margin. No violation. Perfect.

$0 < \xi_i < 1$

Point is inside the margin but on the correct side of the decision boundary. Margin violation.

$\xi_i \geq 1$

Point is misclassified — on the wrong side of the decision boundary. A true error.

The C Hyperparameter (Regularisation)

$C$ controls the trade-off between maximising the margin and minimising violations. It is the most important hyperparameter to tune in SVM.

Large C

Strongly penalise misclassifications → small margin, few violations. Approaches hard-margin SVM. Risk of overfitting.

C → ∞: hard margin

Small C

Allow many violations → large margin, more misclassifications. More flexibility. Risk of underfitting.

C → 0: all points ignored

🔑 C vs λ (Regularization)
Note: $C = \frac{1}{\lambda}$ where $\lambda$ is the regularization strength from regression. Large C = weak regularization (fits data tightly). Small C = strong regularization (allows violations). This is the opposite convention from $\lambda$.

Complete Objective Function (From Lecture)

Soft Margin SVM
Complete Primal Objective (C-SVM)
$$\min_{\mathbf{w}, b, \boldsymbol{\xi}} \frac{1}{2}\|\mathbf{w}\|^2 + C\sum_{i=1}^{m}\xi_i$$ $$\text{subject to: } y_i(\mathbf{w}^T\mathbf{x}_i + b) \geq 1 - \xi_i, \quad \xi_i \geq 0 \quad \forall i$$
TermMeaningEffect
$\frac{1}{2}\|\mathbf{w}\|^2$ Margin maximisation Pushes hyperplane away from all points
$C\sum\xi_i$ Penalty for violations Penalises points inside/crossing the margin
$C$ Trade-off parameter Balances margin size vs constraint violations
$\xi_i \geq 0$ Slack variable Measures how much constraint $i$ is violated
🔄

SVM Dual Form

▼

The dual form is crucial for the kernel trick. By solving the dual problem (using Lagrangian duality), we can express the SVM entirely in terms of inner products $\langle \mathbf{x}_i, \mathbf{x}_j \rangle$ — which can be replaced by kernel functions.

Deriving the Dual (From Lecture)

1
Set up Lagrangian with multipliers $\alpha_i \geq 0$ and $\mu_i \geq 0$: $$\mathcal{L} = \frac{1}{2}\|\mathbf{w}\|^2 + C\sum\xi_i - \sum\alpha_i[y_i(\mathbf{w}^T\mathbf{x}_i+b)-1+\xi_i] - \sum\mu_i\xi_i$$
2
KKT stationarity conditions (set partial derivatives to zero): $$\frac{\partial\mathcal{L}}{\partial\mathbf{w}} = 0 \Rightarrow \mathbf{w} = \sum_{i=1}^m \alpha_i y_i \mathbf{x}_i$$ $$\frac{\partial\mathcal{L}}{\partial b} = 0 \Rightarrow \sum_{i=1}^m \alpha_i y_i = 0$$ $$\frac{\partial\mathcal{L}}{\partial\xi_i} = 0 \Rightarrow C - \alpha_i - \mu_i = 0 \Rightarrow \alpha_i \leq C$$
3
Substitute back into Lagrangian to get the dual — a function only of $\alpha_i$.
Dual Problem
SVM Dual Optimisation
$$\max_{\boldsymbol{\alpha}} \sum_{i=1}^m \alpha_i - \frac{1}{2}\sum_{i=1}^m\sum_{j=1}^m \alpha_i\alpha_j y_i y_j \langle\mathbf{x}_i, \mathbf{x}_j\rangle$$ $$\text{subject to: } \sum_{i=1}^m \alpha_i y_i = 0, \quad 0 \leq \alpha_i \leq C$$
Prediction
Decision function using dual variables
$$\hat{y} = \text{sign}\left(\sum_{i=1}^m \alpha_i y_i \langle\mathbf{x}_i, \mathbf{x}\rangle + b\right)$$
🔑 The Key Insight
Both the dual objective and the prediction function depend on data only through inner products $\langle \mathbf{x}_i, \mathbf{x}_j \rangle$. This is what the kernel trick exploits!
Alpha Values
  • $\alpha_i = 0$: Point is correctly classified outside margin → NOT a support vector
  • $0 < \alpha_i < C$: Point is exactly on the margin boundary → support vector
  • $\alpha_i = C$: Point violates the margin (inside or misclassified) → bounded support vector
✨

The Kernel Trick

▼

Many real datasets are not linearly separable in the original feature space. The kernel trick allows SVM to find non-linear decision boundaries by implicitly mapping data to a higher-dimensional space — without ever computing the mapping explicitly.

Feature Map $\phi$

Idea: map data from original space to a higher-dimensional space where it is linearly separable. Then run the linear SVM in that space.

Feature Map
Mapping to higher-dimensional space
$$\phi: \mathbb{R}^d \to \mathbb{R}^D \quad (D \gg d, \text{ possibly } D = \infty)$$ $$\mathbf{x} \mapsto \phi(\mathbf{x})$$
⚠️ Problem with explicit mapping
If $D$ is very large (or infinite, like with RBF), computing $\phi(\mathbf{x})$ is computationally impossible. But the dual SVM only needs $\langle\phi(\mathbf{x}_i), \phi(\mathbf{x}_j)\rangle$ — the inner product. The kernel trick computes this cheaply!

Kernel Definition

Kernel Function
Computes inner product in feature space
$$K(\mathbf{x}_i, \mathbf{x}_j) = \langle\phi(\mathbf{x}_i), \phi(\mathbf{x}_j)\rangle$$
🔑 The Trick
We replace every inner product $\langle\mathbf{x}_i, \mathbf{x}_j\rangle$ in the dual with $K(\mathbf{x}_i, \mathbf{x}_j)$. The kernel function computes the dot product in the high-dimensional space cheaply in the original space. No explicit $\phi(\mathbf{x})$ needed!
Mercer's Condition
A function $K(\mathbf{x}_i, \mathbf{x}_j)$ is a valid kernel if and only if the kernel matrix $K$ (with $K_{ij} = K(\mathbf{x}_i, \mathbf{x}_j)$) is symmetric positive semi-definite.

SVM with Kernel (From Lecture)

Kernelized SVM Dual
Replace inner products with kernel
$$\max_{\boldsymbol{\alpha}} \sum_i\alpha_i - \frac{1}{2}\sum_i\sum_j\alpha_i\alpha_j y_i y_j K(\mathbf{x}_i, \mathbf{x}_j)$$
Prediction
Kernelized decision function
$$\hat{y} = \text{sign}\left(\sum_{i=1}^m \alpha_i y_i K(\mathbf{x}_i, \mathbf{x}) + b\right)$$
📐

Common Kernels (From Lecture)

▼
Linear Kernel
$$K(\mathbf{x}_i, \mathbf{x}_j) = \mathbf{x}_i^T\mathbf{x}_j$$
When to use
  • Linearly separable data
  • High-dimensional sparse data (e.g., text classification)
  • Large datasets — fastest to compute
  • Same as standard linear SVM (no mapping)
Polynomial Kernel
$$K(\mathbf{x}_i, \mathbf{x}_j) = (\mathbf{x}_i^T\mathbf{x}_j + c)^d$$

$d$ = degree, $c \geq 0$ = constant (often $c=1$)

Math Example (From Lecture)

For $d=2$, $c=0$, $\mathbf{x} = [x_1, x_2]^T$:

$K(\mathbf{x}, \mathbf{z}) = (\mathbf{x}^T\mathbf{z})^2 = (x_1z_1 + x_2z_2)^2$

$= x_1^2z_1^2 + 2x_1x_2z_1z_2 + x_2^2z_2^2$

$= \langle [x_1^2, \sqrt{2}x_1x_2, x_2^2], [z_1^2, \sqrt{2}z_1z_2, z_2^2] \rangle$

So $\phi(\mathbf{x}) = [x_1^2, \sqrt{2}x_1x_2, x_2^2]$ — computes 3D inner product with just a 2D dot product!

When to use
Natural for data with polynomial relationships. Image classification. Degree $d$ is a hyperparameter (usually 2 or 3).
RBF / Gaussian Kernel
$$K(\mathbf{x}_i, \mathbf{x}_j) = \exp\left(-\frac{\|\mathbf{x}_i - \mathbf{x}_j\|^2}{2\sigma^2}\right) = \exp\left(-\gamma\|\mathbf{x}_i - \mathbf{x}_j\|^2\right)$$

$\gamma = \frac{1}{2\sigma^2}$ — controls the "width" of the Gaussian

🔑 RBF Facts
  • Maps to infinite-dimensional feature space! Yet computed in O(d) time.
  • $K(\mathbf{x}_i, \mathbf{x}_j) \in (0, 1]$ — always positive, 1 when $\mathbf{x}_i = \mathbf{x}_j$
  • Default choice when no prior knowledge about data shape
  • Very powerful but needs careful tuning of $\gamma$ and $C$

Effect of $\gamma$ (inverse of bandwidth)

Large $\gamma$ (small $\sigma$)

Narrow Gaussian — each SV influences a very small region. Very wiggly decision boundary. Risk of overfitting.

Small $\gamma$ (large $\sigma$)

Wide Gaussian — each SV influences a large region. Smooth decision boundary. Risk of underfitting.

Sigmoid Kernel
$$K(\mathbf{x}_i, \mathbf{x}_j) = \tanh(\kappa \mathbf{x}_i^T\mathbf{x}_j + c)$$
⚠️ Note
The sigmoid kernel is not always positive semi-definite (not always a valid Mercer kernel). Used less commonly. Similar to a 2-layer neural network.
KernelFormulaParametersBest For
Linear $\mathbf{x}_i^T\mathbf{x}_j$ None Text, linearly separable
Polynomial $(\mathbf{x}_i^T\mathbf{x}_j + c)^d$ $d, c$ Image recognition
RBF/Gaussian $\exp(-\gamma\|\mathbf{x}_i-\mathbf{x}_j\|^2)$ $\gamma$ (or $\sigma$) General purpose, default choice
Sigmoid $\tanh(\kappa\mathbf{x}_i^T\mathbf{x}_j + c)$ $\kappa, c$ Neural network analogy

Week 4 Quiz

▼

🧩 Test Your Understanding

Q1. A slack variable $\xi_i = 0.7$. This point is:

$0 < \xi_i < 1$ means the point is inside the margin but still on the correct side. $\xi_i \geq 1$ would mean misclassification.

Q2. In soft margin SVM, using a very large C value results in:

Large C strongly penalises margin violations, forcing the SVM to find a perfect or near-perfect separator — equivalent to hard-margin SVM when C→∞.

Q3. The kernel trick works because the SVM dual problem depends on data only through:

The dual objective and prediction function only use data through inner products. By replacing $\langle \mathbf{x}_i, \mathbf{x}_j \rangle$ with $K(\mathbf{x}_i, \mathbf{x}_j)$, we implicitly work in a higher-dimensional space.

Q4. The RBF kernel maps data to:

The RBF/Gaussian kernel implicitly corresponds to an infinite-dimensional feature map (proven by Taylor expansion of the exponential). Yet computing $K(\mathbf{x}_i, \mathbf{x}_j)$ only takes O(d) time!

Q5. In the dual SVM, a training point has $\alpha_i = C$. What does this mean?

In the soft-margin dual: $\alpha_i = 0$ (not SV), $0 < \alpha_i < C$ (on margin boundary, perfect SV), $\alpha_i = C$ (violates margin — the point has $\xi_i > 0$).

Q6. For polynomial kernel $K(\mathbf{x}, \mathbf{z}) = (\mathbf{x}^T\mathbf{z})^2$ with $\mathbf{x}=[1,2]$ and $\mathbf{z}=[3,4]$, compute $K$:

$K(\mathbf{x}, \mathbf{z}) = (\mathbf{x}^T\mathbf{z})^2 = (1×3 + 2×4)^2 = (3+8)^2 = 11^2 = 121$

Numerical Problems

▼

Problem 1 — Kernel Computation (Polynomial)

Using the polynomial kernel $K(\mathbf{x}, \mathbf{z}) = (\mathbf{x}^T\mathbf{z} + 1)^2$, compute $K(\mathbf{x}, \mathbf{z})$ for $\mathbf{x} = [2, 3]^T$ and $\mathbf{z} = [1, 4]^T$. Also write out the implied feature map $\phi(\mathbf{x})$.
Step 1
$\mathbf{x}^T\mathbf{z} = 2(1) + 3(4) = 2 + 12 = 14$
Step 2
$K(\mathbf{x}, \mathbf{z}) = (14 + 1)^2 = 15^2 = \mathbf{225}$
Feature Map
Expand $(x_1z_1 + x_2z_2 + 1)^2 = x_1^2z_1^2 + x_2^2z_2^2 + 1 + 2x_1x_2z_1z_2 + 2x_1z_1 + 2x_2z_2$
So $\phi(\mathbf{x}) = [x_1^2, x_2^2, 1, \sqrt{2}x_1x_2, \sqrt{2}x_1, \sqrt{2}x_2]$
$\phi([2,3]) = [4, 9, 1, \sqrt{2}(6), \sqrt{2}(2), \sqrt{2}(3)] = [4, 9, 1, 6\sqrt{2}, 2\sqrt{2}, 3\sqrt{2}]$
Verify: $\langle\phi(\mathbf{x}), \phi(\mathbf{z})\rangle = 4(1)+9(16)+1+72+2(2\sqrt{2})^2... = 225$ ✅

Problem 2 — RBF Kernel Computation

Compute the RBF kernel $K(\mathbf{x}, \mathbf{z}) = \exp(-\gamma\|\mathbf{x}-\mathbf{z}\|^2)$ with $\gamma = 0.5$ for:
(a) $\mathbf{x} = [1,1]^T$, $\mathbf{z} = [1,1]^T$ (same point)
(b) $\mathbf{x} = [0,0]^T$, $\mathbf{z} = [2,2]^T$
(c) $\mathbf{x} = [0,0]^T$, $\mathbf{z} = [10,10]^T$
Part (a)
$\|\mathbf{x}-\mathbf{z}\|^2 = 0$
$K = \exp(-0.5 \times 0) = e^0 = \mathbf{1.0}$
RBF is always 1 when two points are identical ✅
Part (b)
$\|\mathbf{x}-\mathbf{z}\|^2 = (0-2)^2 + (0-2)^2 = 4+4 = 8$
$K = \exp(-0.5 \times 8) = e^{-4} \approx \mathbf{0.0183}$
Part (c)
$\|\mathbf{x}-\mathbf{z}\|^2 = 100+100 = 200$
$K = \exp(-0.5 \times 200) = e^{-100} \approx \mathbf{3.7 \times 10^{-44}}$ (essentially 0)
Far-apart points have near-zero kernel value → don't influence each other's classification.

Problem 3 — Classify with Soft Margin SVM

A soft-margin SVM (linear kernel) has been trained with 3 support vectors:
$\mathbf{x}_1 = [1,0]$, $y_1=+1$, $\alpha_1=0.5$
$\mathbf{x}_2 = [0,1]$, $y_2=+1$, $\alpha_2=0.3$
$\mathbf{x}_3 = [2,2]$, $y_3=-1$, $\alpha_3=0.8$
Bias $b = -0.5$. Classify new point $\mathbf{x}^* = [1,1]$.
Formula
$f(\mathbf{x}^*) = \sum_i \alpha_i y_i K(\mathbf{x}_i, \mathbf{x}^*) + b$ with linear kernel $K(\mathbf{x}_i, \mathbf{x}^*) = \mathbf{x}_i^T\mathbf{x}^*$
Kernel Values
$K(\mathbf{x}_1, \mathbf{x}^*) = [1,0]\cdot[1,1] = 1$
$K(\mathbf{x}_2, \mathbf{x}^*) = [0,1]\cdot[1,1] = 1$
$K(\mathbf{x}_3, \mathbf{x}^*) = [2,2]\cdot[1,1] = 4$
Decision
$f = (0.5)(+1)(1) + (0.3)(+1)(1) + (0.8)(-1)(4) + (-0.5)$
$f = 0.5 + 0.3 - 3.2 - 0.5 = \mathbf{-2.9}$
$\hat{y} = \text{sign}(-2.9) = \mathbf{-1}$ ✅ → Classified as Class -1

Kernel Selection Guide

  • Start with Linear — fast, interpretable
  • Data has unknown shape → try RBF first
  • Known polynomial relationships → Polynomial
  • Always tune $C$ (and $\gamma$ for RBF) via cross-validation

Tuning SVM

  • C: controls margin vs violations (key!)
  • $\gamma$ (RBF): controls locality of influence
  • $d$ (Poly): degree of polynomial
  • Use grid search + cross-validation
