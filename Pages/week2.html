Linear & Polynomial Regression

From fitting a straight line to understanding why models fail — learn regression, cost minimisation, polynomial fitting, and regularisation.

📈

Linear Regression

▼

Linear Regression models the relationship between input feature(s) $x$ and a continuous output $y$ by fitting a straight line (or hyperplane in higher dimensions). It is the foundation of supervised learning.

Hypothesis Function

Simple Linear
Hypothesis — 1 Feature
$$h_\theta(x) = \theta_0 + \theta_1 x$$
Multiple Linear
Hypothesis — Multiple Features
$$h_\theta(\mathbf{x}) = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \ldots + \theta_n x_n = \boldsymbol{\theta}^T \mathbf{x}$$

$\theta_0$ (bias)

The y-intercept — value of $h$ when all features are 0.

$\theta_1$ (weight)

The slope — how much $h$ changes for a unit increase in $x_1$.

$\hat{y}$ vs $y$

$\hat{y} = h_\theta(x)$ is the predicted value; $y$ is the actual label.

Cost Function (Mean Squared Error)

We need to measure how wrong our predictions are. The cost function $J(\theta)$ quantifies this error. We want to minimise $J(\theta)$ to get the best fit line.

Cost Function
Mean Squared Error (MSE)
$$J(\theta_0, \theta_1) = \frac{1}{2m} \sum_{i=1}^{m} \left(h_\theta(x^{(i)}) - y^{(i)}\right)^2$$

$m$ = number of training examples. The $\frac{1}{2}$ makes the gradient cleaner.

🔑 Why Squared?
Squaring makes all errors positive, penalises large errors more (quadratic penalty), and is differentiable everywhere — perfect for gradient descent.
📌 Other Names
MSE = Mean Squared Error. When divided by $\frac{1}{2m}$ as above, this is also called the "Squared Error Loss". The equivalent without the $\frac{1}{2}$ is just MSE. RSS (Residual Sum of Squares) = $\sum(y_i - \hat{y}_i)^2$.

Gradient Descent for Linear Regression

We derive the partial derivatives of the cost function and plug them into the GD update rule:

Derivatives
Partial derivatives of MSE cost
$$\frac{\partial J}{\partial \theta_0} = \frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)}) - y^{(i)})$$ $$\frac{\partial J}{\partial \theta_1} = \frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)}) - y^{(i)}) \cdot x^{(i)}$$
Update Rules
Simultaneous update of parameters
$$\theta_0 := \theta_0 - \alpha \cdot \frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)}) - y^{(i)})$$ $$\theta_1 := \theta_1 - \alpha \cdot \frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)}) - y^{(i)}) \cdot x^{(i)}$$
✅ Convex Function
The MSE cost function for linear regression is convex — it has a single global minimum. Gradient descent will always converge (with appropriate $\alpha$). No local minima to worry about!

Normal Equation — Closed Form Solution

Instead of iterative gradient descent, we can solve for $\theta$ analytically in one step using the Normal Equation.

Normal Equation
Closed-form optimal parameters
$$\boldsymbol{\theta} = (\mathbf{X}^T \mathbf{X})^{-1} \mathbf{X}^T \mathbf{y}$$

Polynomial Regression

▼

When data doesn't follow a straight line, we extend linear regression by adding polynomial (higher-degree) terms of the original features. It's still linear in the parameters $\theta$!

Model
Polynomial Regression of degree d
$$h_\theta(x) = \theta_0 + \theta_1 x + \theta_2 x^2 + \ldots + \theta_d x^d$$
💡 Still Linear Regression!
If we define new features $x_1 = x$, $x_2 = x^2$, $x_3 = x^3$, then $h = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \theta_3 x_3$. This is just multiple linear regression with engineered features!
📋 Lecture Example: Hardwood Dataset

19 observations of kraft paper strength $y$ vs hardwood percentage $x$. A quadratic model $y = \theta_0 + \theta_1 x + \theta_2 x^2$ captures the non-linear relationship better than a simple line.

Important Considerations (From Lecture)

1
Order of the Model Always possible to fit degree $n-1$ polynomial to $n$ points perfectly — but this almost surely means overfitting! Use parsimony: simplest model consistent with the data.
2
Model Building Strategies
  • Forward Selection: Start with degree 1, add higher terms until the highest-order term's t-test becomes non-significant.
  • Backward Elimination: Start with highest degree, remove highest-order terms one by one until the remaining highest term is significant.

⚠️ These two methods don't always give the same model!

3
Extrapolation Danger Polynomial models can turn in unanticipated directions outside the training range. Extrapolation with polynomials is extremely hazardous.
4
Hierarchy Principle If you include a higher-degree term (e.g., $x^2$), you should also include all lower-degree terms (e.g., $x$). Always respect the hierarchy.
5
Ill-Conditioning High-degree polynomials can cause numerical instability (ill-conditioned $\mathbf{X}^T\mathbf{X}$ matrix). Consider orthogonal polynomials for high degrees.

Bias-Variance Tradeoff

▼

Every ML model makes two types of errors. Understanding this tradeoff is crucial — it explains why complex models don't always win, and why simple models don't always lose.

Bias (Underfitting)

Error from wrong assumptions in the model. A high-bias model is too simple — it can't capture the true pattern even on training data.

Example: Using a straight line to fit curved data.

Variance (Overfitting)

Error from sensitivity to fluctuations in training data. A high-variance model is too complex — it memorises noise instead of the true pattern.

Example: Degree-9 polynomial to fit 10 noisy points.

Mathematical Decomposition
Expected Test Error Decomposition
$$\text{Expected Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$ $$E\left[(y - \hat{f}(x))^2\right] = \left[\text{Bias}(\hat{f}(x))\right]^2 + \text{Var}(\hat{f}(x)) + \sigma^2$$
🔑 Key Insight
Irreducible noise $\sigma^2$ is the inherent randomness in data — we can never eliminate it. We can only optimise Bias² + Variance. Reducing one often increases the other.

Underfitting (High Bias)

Signs of Underfitting
  • High training error AND high test error
  • Model is too simple for the data complexity
  • Adding more training data doesn't help much

Fix: Use a more complex model (higher degree polynomial, more features, fewer regularization constraints).

Overfitting (High Variance)

Signs of Overfitting
  • Very low training error but HIGH test error
  • Large gap between training and test performance
  • Model memorised the training data including noise

Fix: Get more data, reduce model complexity, apply regularization, use cross-validation.

Bias-Variance Curve

As model complexity increases, Bias ↓ but Variance ↑

🔵 Bias² | 🔴 Variance | 🟡 Total Error | ★ Sweet Spot (optimal complexity)

Regularization

▼

Regularization adds a penalty term to the cost function to discourage large parameter values. This prevents overfitting by keeping the model simpler.

General Form
Regularized Cost Function
$$J(\theta) = \underbrace{\frac{1}{2m}\sum_{i=1}^m(h_\theta(x^{(i)}) - y^{(i)})^2}_{\text{Data fit (MSE)}} + \underbrace{\lambda \cdot \Omega(\theta)}_{\text{Regularization penalty}}$$

$\lambda$ (lambda) = regularization strength. Higher $\lambda$ = stronger penalty = simpler model.

L2 Penalty
Ridge Regression Cost Function
$$J(\theta) = \frac{1}{2m}\sum_{i=1}^m(h_\theta(x^{(i)}) - y^{(i)})^2 + \lambda\sum_{j=1}^n \theta_j^2$$
✅ Ridge Properties
  • Penalises squared magnitudes of coefficients
  • Shrinks all coefficients towards zero — but never exactly zero
  • Keeps all features (just makes them small)
  • Works well when all features are relevant
  • Has a closed-form solution: $\theta = (X^TX + \lambda I)^{-1}X^Ty$
L1 Penalty
Lasso Regression Cost Function
$$J(\theta) = \frac{1}{2m}\sum_{i=1}^m(h_\theta(x^{(i)}) - y^{(i)})^2 + \lambda\sum_{j=1}^n |\theta_j|$$
✅ Lasso Properties
  • Penalises absolute magnitudes of coefficients
  • Can shrink coefficients to exactly zero — performs feature selection!
  • Produces sparse models (many zeros)
  • Best when only a subset of features matter
  • No closed-form — needs iterative methods (coordinate descent)

ElasticNet
Combines both: $\lambda_1\sum|\theta_j| + \lambda_2\sum\theta_j^2$. Gets benefits of both Lasso (sparsity) and Ridge (stability).
⚠️ Don't regularize $\theta_0$
The bias term $\theta_0$ is conventionally excluded from the regularization penalty. Only $\theta_1, \ldots, \theta_n$ are penalised.

Effect of $\lambda$

$\lambda \to 0$

Regularization has no effect. Standard regression. Risk of overfitting.

$\lambda \to \infty$

All $\theta_j \to 0$ (only bias remains). Extreme underfitting — just predicts the mean.

Week 2 Quiz

▼

🧩 Test Your Understanding

Q1. The cost function for linear regression is convex. What does this mean for gradient descent?

Convex functions have a single global minimum. Gradient descent on a convex function with an appropriate learning rate is guaranteed to converge to the global minimum.

Q2. A model has Training Error = 2%, Test Error = 40%. What is the problem?

Low training error but high test error is the classic sign of overfitting (high variance). The model memorised training data but generalises poorly.

Q3. Which regularisation technique can set coefficients exactly to zero, performing feature selection?

Lasso (L1) uses absolute value penalties which can force coefficients to exactly zero, effectively removing features from the model. Ridge (L2) only shrinks them close to zero.

Q4. In the Normal Equation $\theta = (X^TX)^{-1}X^Ty$, what is the computational bottleneck?

Matrix inversion is O(n³) where n is the number of features. For large feature counts (n > 10,000), this becomes prohibitively slow and gradient descent is preferred.

Q5. Forward selection in polynomial regression means:

Forward selection starts simple and adds complexity. Backward elimination starts complex and removes. From the lecture: "Successively fit models of increasing order until the t test for the highest order term is nonsignificant."

Numerical Problems

Problem 1 — Compute MSE Cost Function

Given training data: $x = [1, 2, 3]$, $y = [2, 4, 6]$. Current model: $h_\theta(x) = 0.5 + 1.5x$ (so $\theta_0=0.5$, $\theta_1=1.5$). Compute the MSE cost $J(\theta)$.
Step 1
Compute predictions $\hat{y}_i = 0.5 + 1.5x_i$:
$\hat{y}_1 = 0.5 + 1.5(1) = 2.0$
$\hat{y}_2 = 0.5 + 1.5(2) = 3.5$
$\hat{y}_3 = 0.5 + 1.5(3) = 5.0$
Step 2
Compute residuals $(y_i - \hat{y}_i)$:
$(2 - 2.0)^2 = 0$
$(4 - 3.5)^2 = 0.25$
$(6 - 5.0)^2 = 1.0$
Step 3
$J(\theta) = \frac{1}{2m}\sum = \frac{1}{2 \times 3}(0 + 0.25 + 1.0) = \frac{1.25}{6} \approx \mathbf{0.208}$

Problem 2 — One Step of Gradient Descent for Linear Regression

Training data: $(x^{(1)},y^{(1)}) = (1,3)$, $(x^{(2)},y^{(2)}) = (2,5)$.
Initial: $\theta_0 = 0$, $\theta_1 = 0$, $\alpha = 0.1$.
Perform one update step of gradient descent.
Predictions
$h_\theta(1) = 0 + 0(1) = 0$, $h_\theta(2) = 0 + 0(2) = 0$
Errors
$(h-y)_1 = 0 - 3 = -3$, $(h-y)_2 = 0 - 5 = -5$
Gradients
$\frac{\partial J}{\partial \theta_0} = \frac{1}{2}[(-3) + (-5)] = \frac{-8}{2} = -4$
$\frac{\partial J}{\partial \theta_1} = \frac{1}{2}[(-3)(1) + (-5)(2)] = \frac{-13}{2} = -6.5$
Update
$\theta_0 := 0 - 0.1(-4) = \mathbf{0.4}$
$\theta_1 := 0 - 0.1(-6.5) = \mathbf{0.65}$

Problem 3 — Ridge vs Lasso Effect on Coefficients

A linear regression model has coefficients $\theta_1 = 3.0$ and $\theta_2 = 0.1$. With regularization parameter $\lambda = 1$ and only 1 data point (simplified), which regularization would likely drive $\theta_2$ to zero?
Ridge Penalty
Ridge adds $\lambda(\theta_1^2 + \theta_2^2) = 1(9 + 0.01) = 9.01$ to cost.
Ridge gradient: $\nabla_{\theta_2} = \ldots + 2\lambda\theta_2 = \ldots + 0.2$. Shrinks towards 0 but never reaches exactly 0.
Lasso Penalty
Lasso adds $\lambda(|\theta_1| + |\theta_2|) = 1(3.0 + 0.1) = 3.1$ to cost.
Lasso uses subgradient $\text{sign}(\theta_j)$. For small $|\theta_2| = 0.1$, the lasso penalty can overcome the data fit gradient, driving $\theta_2$ exactly to 0.
Answer
✅ Lasso (L1) would drive $\theta_2$ to exactly zero — performing automatic feature selection. Ridge would only shrink it to a small value.
