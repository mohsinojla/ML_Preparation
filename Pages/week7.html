Week 7

Hierarchical Clustering

Move beyond flat partitioning — discover how agglomerative clustering builds a dendrogram tree that reveals cluster structure at every scale, without needing to specify K in advance.

🌳

Limitations of Flat Algorithms

▼

K-Means and similar partition algorithms produce a flat clustering — a single assignment of every point to exactly one cluster. This has key limitations:

⚠ Problem 1: Must specify K
You must decide the number of clusters before running the algorithm. Wrong K gives meaningless results.
⚠ Problem 2: No hierarchy
No information about relationships between clusters or sub-cluster structure.
✅ Hierarchical Clustering Solution
Builds a tree of clusters (dendrogram) that captures structure at every level. K can be chosen after the algorithm finishes by cutting the tree at any height.
⚙

Agglomerative Algorithm

▼

Agglomerative (bottom-up) is the most common approach. Start with every point as its own cluster, then repeatedly merge the two closest clusters.

1
Initialize: Assign each of N data points to its own cluster. You have N clusters.
2
Compute distances: Build an N×N distance matrix between all cluster pairs.
3
Merge closest: Find the two clusters with minimum distance. Merge them into one new cluster.
4
Update distances: Recompute distances from the new merged cluster to all remaining clusters (using chosen linkage method).
5
Repeat: Go back to step 3 until only 1 cluster remains.
6
Cut dendrogram: Choose a height to cut the tree to obtain K clusters.
🔗

Linkage Methods

▼

When merging, we need a distance metric between clusters (not just points). The three main options:

Single Linkage (Minimum):

$$d(C_1, C_2) = \min_{x \in C_1,\, y \in C_2} d(x, y)$$

Complete Linkage (Maximum):

$$d(C_1, C_2) = \max_{x \in C_1,\, y \in C_2} d(x, y)$$

Average Linkage (UPGMA):

$$d(C_1, C_2) = \frac{1}{|C_1||C_2|} \sum_{x \in C_1} \sum_{y \in C_2} d(x, y)$$
LinkageDistance usedCluster shapeSensitivity
SingleMinimum pairwiseLong, chain-likeOutlier sensitive (chaining effect)
CompleteMaximum pairwiseCompact, sphericalRobust to outliers
AverageMean of all pairsBalancedModerate
📌 Which to use?
Complete linkage is preferred for compact clusters. Single linkage is good for elongated or irregular shapes. Average is a safe default.
📊

Dendrogram

▼

A dendrogram is a tree diagram that records the entire merge history. The x-axis shows the data points and the y-axis shows the distance (height) at which merges occurred.

🔑 Reading the Dendrogram
  • Leaves at bottom = individual data points
  • Height of each horizontal bar = distance when those clusters merged
  • A horizontal cut at height h gives clusters = connected components below h
  • Tall vertical lines = large gap between merges = natural cluster boundary
✂ Choosing K from the Dendrogram
Look for the largest vertical gap (where the dendrogram bars are tallest before the next merge). Cut there. The number of vertical lines you cross = K.
📐

Worked Example: {18, 22, 25, 42, 27, 43}

▼

Points on a 1D number line: 18, 22, 25, 27, 42, 43 (sorted). Using Single Linkage with absolute difference as distance.

Initial distances (sorted pairs):
|18−22|=4, |22−25|=3, |25−27|=2, |27−42|=15, |42−43|=1
M1
Merge 42 and 43 — distance = 1 (smallest). New cluster: {42,43}
M2
Merge 25 and 27 — distance = 2. New cluster: {25,27}
M3
Merge 22 and {25,27} — single link dist = min(|22−25|,|22−27|)=3. New cluster: {22,25,27}
M4
Merge 18 and {22,25,27} — single link dist = |18−22|=4. New cluster: {18,22,25,27}
M5
Merge {18,22,25,27} and {42,43} — single link dist = min(|27−42|,|27−43|)=15. Final single cluster.
✂ Cut at height ~10
We get 2 clusters: {18,22,25,27} and {42,43}. The large gap (distance 15) between M4 and M5 is the natural cluster boundary.
🎯

Discovering K from the Dendrogram

▼

The key advantage of hierarchical clustering: you do not need to specify K before running the algorithm. The dendrogram encodes all possible K values.

1
Identify the longest vertical line in the dendrogram (biggest gap between merge heights).
2
Draw a horizontal line through that gap without crossing any horizontal merge bar.
3
Count how many vertical lines the horizontal cut intersects. That count = K.
📌 Domain knowledge
The dendrogram suggests K, but domain knowledge is still valuable. The dendrogram shows you where natural breaks occur at multiple granularities simultaneously.
⚖

Quiz — Hierarchical Clustering

▼

Q1. In agglomerative clustering, what is the starting state?

Agglomerative means bottom-up. We start with N clusters (each point alone) and merge until 1 cluster remains.

Q2. Single linkage distance between two clusters is defined as:

Single linkage uses the minimum (closest pair) distance. This can cause chaining — long elongated clusters.

Q3. In the example {18,22,25,42,27,43} with single linkage, which two points merge first?

42 and 43 have distance 1 — the smallest pairwise distance, so they merge first.

Q4. How do you determine K from a dendrogram?

The largest vertical gap in the dendrogram indicates a natural cluster boundary. A horizontal cut there, counting vertical line crossings, gives K.

Q5. Complete linkage tends to produce:

Complete linkage uses the maximum distance, which prevents the chaining effect and tends to produce more compact clusters.

Q6. What is the time complexity of agglomerative hierarchical clustering?

The O(n² log n) complexity makes hierarchical clustering impractical for very large datasets compared to K-Means.

Numerical Problems

▼
Problem 1 — Single Linkage Step-by-Step
Apply agglomerative clustering with single linkage to points {18, 22, 25, 42, 27, 43}. Show all merge steps and the final dendrogram height for each merge.
Step 1Sort: 18, 22, 25, 27, 42, 43. Compute all pairwise distances (1D absolute difference).
Step 2Smallest distance: |42−43|=1. Merge {42} and {43} → cluster A={42,43} at height 1.
Step 3Next smallest: |25−27|=2. Merge {25} and {27} → cluster B={25,27} at height 2.
Step 4Next: {22} to B={25,27}: single link = min(|22−25|,|22−27|)=3. Merge → cluster C={22,25,27} at height 3.
Step 5Next: {18} to C: single link = min(|18−22|,|18−25|,|18−27|)=4. Merge → cluster D={18,22,25,27} at height 4.
Step 6Final merge: D={18,22,25,27} to A={42,43}: single link = |27−42|=15. Merge at height 15.
ResultCut at height ~10 gives 2 clusters: {18,22,25,27} and {42,43}. The gap of 11 (from h=4 to h=15) is the natural boundary.
Problem 2 — Complete Linkage Distance
After merge M2, we have clusters C1={18,22} and C2={25,27,42}. Compute the complete linkage distance between C1 and C2.
FormulaComplete linkage: $d(C_1,C_2) = \max_{x \in C_1, y \in C_2} |x-y|$
Pairs|18−25|=7, |18−27|=9, |18−42|=24, |22−25|=3, |22−27|=5, |22−42|=20
AnswerMax = 24. Complete linkage distance = 24.
Problem 3 — Average Linkage Distance
Compute the average linkage distance between cluster C1={18,22} and C2={42,43}.
FormulaAverage linkage: $d = \frac{1}{|C_1||C_2|}\sum_{x\in C_1}\sum_{y\in C_2}|x-y|$
Compute|18−42|=24, |18−43|=25, |22−42|=20, |22−43|=21. Sum = 24+25+20+21 = 90.
Answer$d = 90 / (2 \times 2) = 22.5$
