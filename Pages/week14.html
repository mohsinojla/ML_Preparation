📐

KS Test — Deep Dive

▼

The Kolmogorov-Smirnov (KS) test is a non-parametric test that compares two distributions by measuring the maximum absolute difference between their empirical CDFs.

$$D = \sup_x |F_{\text{train}}(x) - F_{\text{prod}}(x)|$$ where $F(x)$ is the empirical CDF: $F(x) = \frac{\text{number of samples} \leq x}{n}$
KS Statistic DInterpretationAction
D < 0.1No meaningful driftMonitor as usual
0.1 ≤ D < 0.2Mild driftIncrease monitoring frequency
D ≥ 0.2Significant driftInvestigate and consider retraining
📌 p-value interpretation
The KS test also produces a p-value. p < 0.05 means statistically significant difference between distributions. Use D for magnitude and p-value for significance.
📊

Three Pillars of Observability

▼
Monitoring
Numbers that trigger alerts. Aggregated metrics: CPU usage, request latency, model accuracy rate, prediction distribution. "CPU is at 99%."
Logging
Granular timestamped records for debugging. "Line 42 crashed because user_id was null at 2024-01-15 14:23:07." Structured logs in ELK stack.
Distributed Tracing
Track a request across microservices. Shows the full path: API gateway → feature service → model server → database, with latency at each hop.
🔑 Monitoring vs Logging
Monitoring tells you THAT something is wrong (alert fires). Logging tells you WHY it's wrong (debugging). Both are needed.
📝

What to Log in ML Systems

▼
1
Input features — raw features sent to the model. Enables drift detection and debugging of bad inputs.
2
Model outputs — predictions, probabilities, decision scores. Monitor prediction distribution shift.
3
Ground truth (when available) — actual outcomes for computing real performance metrics over time.
4
System metrics — latency, throughput, error rates, CPU/memory usage per inference.
5
Metadata — model version, feature pipeline version, timestamp, request ID for tracing.

Alerting Strategy P1–P4

▼
PrioritySeverityResponse timeExample
P1 — CriticalSystem down / complete failureImmediate (<15 min)Model API returning 500 errors for all requests. Wake engineer immediately.
P2 — HighSignificant degradation<1 hourModel accuracy dropped 20% in last hour. Significant drift detected (D≥0.2).
P3 — MediumMild degradationNext business dayMild drift (D=0.15). Prediction latency at 200ms (SLA is 100ms).
P4 — LowInformationalWeekly reviewFeature importance shift. Gradual accuracy trend downward (<2%).
📌 Alert fatigue
Too many alerts = engineers ignore them. Carefully calibrate thresholds. Use P1/P2 sparingly — only for genuinely critical issues that require immediate action.

Quiz — Observability & Monitoring

▼

Q1. For the lecture example {1,2,3,4} vs {3,4,5,6}, what is the KS statistic D?

D = max|F_ref - F_prod|. At x=2: |F_ref(2)-F_prod(2)| = |0.50-0.00| = 0.50. This is the maximum gap.

Q2. A KS statistic D = 0.15 indicates:

0.1 ≤ D < 0.2 = mild drift. Increase monitoring frequency but not necessarily retrain yet.

Q3. Which observability pillar is used for debugging "line 42 crashed because user_id was null"?

Logging provides granular, timestamped text records for debugging. Monitoring gives aggregate numbers (CPU 99%) but can't tell you why a specific request failed.

Q4. Which tool in the ML observability stack is used specifically for drift detection reports?

Evidently AI is ML-specific: it generates data drift, model performance, and data quality monitoring reports. The other tools are general infrastructure tools.

Q5. Which statistical test is best for detecting drift in categorical features?

KS test requires continuous data. Chi-square compares observed vs expected frequency counts — perfect for categorical features like city, product category, etc.

Q6. A P1 alert in the alerting framework means:

P1 = Critical. System down or complete failure. Requires immediate response (within 15 minutes). Examples: model API returning 500 errors for all requests.

Problem 3 — PSI Calculation
Banking: Credit score buckets. Expected (train): [0.3, 0.4, 0.3] for low/medium/high risk. Actual (prod): [0.5, 0.3, 0.2]. Compute PSI. (PSI = Σ(Actual-Expected)·ln(Actual/Expected))
Bucket 1 (low)(0.5-0.3)·ln(0.5/0.3) = 0.2·ln(1.667) = 0.2·0.511 = 0.1022
Bucket 2 (med)(0.3-0.4)·ln(0.3/0.4) = (-0.1)·ln(0.75) = (-0.1)·(-0.288) = 0.0288
Bucket 3 (high)(0.2-0.3)·ln(0.2/0.3) = (-0.1)·ln(0.667) = (-0.1)·(-0.405) = 0.0405
PSIPSI = 0.1022 + 0.0288 + 0.0405 = 0.1715. PSI > 0.1 = moderate shift. PSI > 0.25 = major shift.
