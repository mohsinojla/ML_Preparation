Week 11

Model Optimization & Feature Engineering

Squeeze maximum performance from your models — master hyperparameter tuning with Optuna, avoid data leakage pitfalls, and encode features correctly including tricky cyclical features.

🎛

Hyperparameter Tuning

▼

Hyperparameters are settings you choose before training: learning rate, number of trees, max depth, regularization strength. They are not learned from data — you must search for the best values.

⚠ Manual Tuning
Expert intuition + trial and error. Slow, biased, not reproducible. Fine for quick experiments.
✅ Automated Tuning
Systematic search: grid search, random search, or Bayesian optimization. Reproducible and thorough.
🔍

Grid vs Random vs Bayesian Search

▼
MethodHow it worksEfficiencyBest for
Grid SearchTry every combination in a defined gridExhaustive but slow — O(N^k) evaluationsFew hyperparameters, small search space
Random SearchSample random combinations uniformlyFast — finds good results with fewer evaluations than grid searchMany hyperparameters, wide ranges
Bayesian (Optuna)Build a surrogate model of the objective, sample where it's likely bestMost efficient — learns from past trialsExpensive evaluations, complex spaces
📌 Key insight (Random vs Grid)
Random search is better than grid search when hyperparameters differ in importance. Grid wastes evaluations testing irrelevant combinations; random covers the important dimensions better.
⚡

Optuna & TPE Algorithm

▼

Optuna is a hyperparameter optimization framework using TPE (Tree-structured Parzen Estimator). It models the distribution of good and bad hyperparameter configurations.

Study
A collection of trials. Represents one optimization problem (e.g., tuning XGBoost).
Trial
One evaluation of the objective function with a specific set of hyperparameters.
Objective Function
Function that takes hyperparams, trains a model, and returns a metric to minimize/maximize.
TPE Key Idea:
Maintain two distributions based on past trials: $$l(x) = \text{distribution of good hyperparams (high performance)}$$ $$g(x) = \text{distribution of bad hyperparams (low performance)}$$ Sample next hyperparameter from the ratio: $\arg\max_x \frac{l(x)}{g(x)}$
📌 Optuna API Concepts
study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=100)
study.best_params — retrieve the best found configuration.
🔄

Cross-Validation

▼

k-fold cross-validation splits data into k folds. Train on k-1 folds, validate on the remaining fold. Repeat k times and average the metric. Gives robust estimate of model performance.

1
Split data into k equal folds (typically k=5 or k=10).
2
Iteration i: train on all folds except fold i, validate on fold i.
3
Repeat for all k folds. Average metric across k iterations = CV score.
⚠ Never preprocess on full dataset before CV
Fitting a scaler or imputer on the full dataset before cross-validation leaks test data statistics into training. Always fit preprocessing inside each fold.
🚰

Data Leakage

▼

Data leakage = inclusion of information about the target variable that would NOT be available at actual prediction time. It causes unrealistically high training/validation scores that don't generalize.

Type 1: Train-Test Contamination
Preprocessing (scaling, imputation, feature selection) is fit on the full dataset including test data. Test statistics leak into training. Fix: fit preprocessing only on training data, then transform test data.
Type 2: Target Leakage
Using features that are collected after the event you're predicting. E.g., using "treatment_received" to predict "got sick" — you only know treatment after diagnosis. Fix: only use features available at prediction time.
🔑 Red flag
If your model's validation accuracy is suspiciously high (near 100%), suspect data leakage. Real-world models rarely achieve near-perfect performance.

Cyclical Feature Encoding

▼

Time features like hour of day (0–23) are cyclical: hour 23 and hour 0 are only 1 hour apart, but numerically they appear 23 apart. Linear models and trees won't understand this.

⚠ Problem with raw encoding
If you encode hour as 0–23, the model sees 23:59 and 00:00 as very different (23 units apart). But they are only 1 minute apart in reality.
Cyclical Encoding with period T: $$x_{\sin} = \sin\!\left(\frac{2\pi \cdot x}{T}\right), \quad x_{\cos} = \cos\!\left(\frac{2\pi \cdot x}{T}\right)$$ For hour of day: T = 24. For day of week: T = 7. For month: T = 12.
✅ Why two features?
Using only sin is ambiguous (sin(x) = sin(π - x)). Together, sin and cos uniquely encode every point on the cycle and preserve the cyclical distance structure.
📌 Hour of day example
Hour 0: sin(0)=0, cos(0)=1. Hour 6: sin(π/2)=1, cos(π/2)=0. Hour 12: sin(π)=0, cos(π)=-1. Hour 23: sin(23·2π/24)≈-0.259, cos(...)≈0.966. Note: hour 0 and hour 23 are close in this encoding!

Quiz — Optimization & Feature Engineering

▼

Q1. Label encoding a nominal feature (e.g., colors: Red=0, Green=1, Blue=2) creates what problem?

Label encoding creates integer ordering. For nominal categories with no order, this misleads the model into thinking Blue (2) > Green (1) > Red (0).

Q2. Bayesian optimization (Optuna/TPE) is more efficient than random search because:

TPE models distributions l(x) and g(x) of good/bad configs. It samples from l(x)/g(x) ratio — focusing on regions likely to yield improvement, unlike random uniform sampling.

Q3. What is target leakage?

Target leakage: using features collected after the event being predicted. Example: using "hospitalized" to predict "fell ill" — you only know hospitalization after illness.

Q4. To encode hour of day (0–23) cyclically with T=24, you compute:

Cyclical encoding: sin(2π·x/T) and cos(2π·x/T). Both sin and cos are needed for a unique representation on the unit circle.

Q5. In target encoding, the city "Lahore" is replaced by:

Target encoding replaces each category with the mean target value for that category. From lecture: Lahore→0.5 (or 0.75 depending on data), representing the average purchase rate for that city.

Q6. Train-test contamination occurs when:

Fitting preprocessing on the full dataset means test data statistics influence the transformation. The scaler "sees" test data before evaluation — a form of data leakage.

Numerical Problems

▼
Problem 1 — Cyclical Encoding for Hours
Compute sin and cos cyclical encodings for hours [0, 6, 12, 18] with period T=24. Round to 3 decimal places.
Formula$x_{\sin} = \sin(2\pi h/24)$, $x_{\cos} = \cos(2\pi h/24)$
h=0$\sin(0) = 0.000$, $\cos(0) = 1.000$
h=6$\sin(\pi/2) = 1.000$, $\cos(\pi/2) = 0.000$
h=12$\sin(\pi) = 0.000$, $\cos(\pi) = -1.000$
h=18$\sin(3\pi/2) = -1.000$, $\cos(3\pi/2) = 0.000$
Distance checkh=23: $\sin(23\times2\pi/24)\approx-0.259$, $\cos\approx0.966$. h=0: (0,1). Euclidean distance ≈ $\sqrt{0.259^2+(1-0.966)^2}\approx0.262$ — very close!
Problem 2 — Target Encoding
City feature with targets (bought=1/0): Lahore:[1,0,1,0], Karachi:[0,0], Multan:[1,1,1]. Compute target encoding for each city.
LahoreMean([1,0,1,0]) = 2/4 = 0.5 → encode all Lahore rows as 0.5
KarachiMean([0,0]) = 0/2 = 0.0 → encode all Karachi rows as 0.0
MultanMean([1,1,1]) = 3/3 = 1.0 → encode all Multan rows as 1.0
CautionApply this encoding within CV folds to avoid target leakage.
Problem 3 — Grid Search Size
Grid search over: learning_rate=[0.01, 0.1, 0.5], max_depth=[3,5,7,10], n_estimators=[100,200,500]. How many total evaluations? If each takes 2 seconds and you run 5-fold CV, how long does it take?
Grid size3 × 4 × 3 = 36 combinations
With 5-fold CV36 × 5 = 180 model fits
Total time180 × 2 seconds = 360 seconds = 6 minutes
InsightFor expensive models (e.g., deep nets taking 10+ min each), grid search becomes infeasible. Use random search or Bayesian optimization instead.
