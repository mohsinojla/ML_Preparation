Week 10

Model Evaluation — Imbalanced Classes

Accuracy is a lie when classes are imbalanced. Learn Precision, Recall, F1-score, SMOTE oversampling, and cost-sensitive learning to handle real-world skewed datasets.

⚠

Why Accuracy Fails

▼

Consider a dataset where 99% of examples are negative (no fraud) and 1% are positive (fraud). A model that predicts everything negative achieves 99% accuracy — but it is completely useless!

⚠ The 99% Accuracy Trap
A "dumb" model that always predicts the majority class can have high accuracy on imbalanced data. Accuracy does not capture whether we catch the rare positive cases that actually matter.
$$\text{Accuracy} = \frac{TP + TN}{TP + FP + TN + FN}$$ For 99% negative class: predict all negative → Accuracy = TN/(TN+FN) = 0.99. But Recall = TP/(TP+FN) = 0/anything = 0.
✅ Solution
Use Precision, Recall, and F1-score instead. These focus on the positive class performance.
📊

Confusion Matrix

▼

A confusion matrix tabulates four types of outcomes for a binary classifier:

Predicted PositivePredicted Negative
Actual PositiveTP — True PositiveFN — False Negative (Miss)
Actual NegativeFP — False Positive (False Alarm)TN — True Negative
Type I Error = FP
Model says positive but it's negative. "False alarm." E.g., blocking a legitimate transaction.
Type II Error = FN
Model says negative but it's positive. "Miss." E.g., missing a fraudulent transaction.
📐

Precision, Recall, and F1

▼
Precision — of all predicted positives, how many are actually positive? $$\text{Precision} = \frac{TP}{TP + FP}$$ Recall (Sensitivity) — of all actual positives, how many did we catch? $$\text{Recall} = \frac{TP}{TP + FN}$$ F1-Score — harmonic mean of Precision and Recall: $$F1 = \frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$
🎯 When to prioritize Precision
When false positives are costly. Example: email spam filter — you don't want to wrongly mark important emails as spam.
🎯 When to prioritize Recall
When false negatives are costly. Example: cancer screening — missing a real cancer case is far worse than a false alarm.
📌 Precision-Recall Trade-off
Increasing the classification threshold increases precision but reduces recall. F1 balances both. Choose based on domain requirements.
🔁

SMOTE — Synthetic Minority Over-sampling

▼

SMOTE (Synthetic Minority Over-sampling TEchnique) creates synthetic minority class samples to balance the dataset, rather than simply duplicating existing minority examples.

SMOTE synthetic sample generation: $$x_{\text{new}} = x_i + \lambda \cdot (x_{\text{neighbor}} - x_i), \quad \lambda \in [0, 1] \text{ random}$$ where $x_i$ is a minority sample and $x_{\text{neighbor}}$ is one of its k-nearest neighbors in the minority class.
1
For each minority class sample $x_i$, find its k-nearest neighbors in the minority class.
2
Randomly select one neighbor $x_{\text{neighbor}}$.
3
Generate $x_{\text{new}} = x_i + \lambda(x_{\text{neighbor}} - x_i)$ with random $\lambda \in [0,1]$. This creates a point along the line between them.
4
Add $x_{\text{new}}$ to the training set as a positive minority example. Repeat until balanced.
✅ Why better than simple duplication
Duplication doesn't add new information — the model just memorizes the same points. SMOTE creates new points in the feature space, helping the model generalize to the minority class.
💰

Cost-Sensitive Learning & Class Weights

▼

Instead of resampling, assign higher misclassification cost to the minority class. The model will try harder not to miss minority class examples.

📌 Class weight formula
A common heuristic: $w_{\text{minority}} = \frac{N_{\text{majority}}}{N_{\text{minority}}}$. If majority class has 990 samples and minority has 10, weight = 99.
Mathematically equivalent
Cost-sensitive learning is equivalent to oversampling the minority class by the weight factor.
sklearn API
Use class_weight='balanced' in sklearn classifiers. Automatically sets weights inversely proportional to class frequencies.

Quiz — Model Evaluation

▼

Q1. A model predicts all examples as negative on a 99% negative dataset. Its accuracy is:

Predicting all negative gives 99% accuracy on a 99% negative dataset but recall = 0. It never detects any true positive — completely useless for fraud detection.

Q2. Precision measures:

Precision = TP/(TP+FP). It answers: "When we predict positive, how often are we right?"

Q3. F1-score is defined as:

F1 = 2·P·R/(P+R). It's the harmonic mean, which is lower than the arithmetic mean and penalizes large imbalances between P and R.

Q4. SMOTE creates new minority samples by:

SMOTE formula: x_new = x_i + λ(x_neighbor - x_i), λ∈[0,1]. This creates synthetic points along the line between existing minority examples.

Q5. Which metric should you prioritize for a cancer screening system?

Missing a real cancer (FN) is far worse than a false alarm (FP). Recall = TP/(TP+FN) — prioritize it to minimize missed cancers.

Q6. Cost-sensitive learning is mathematically equivalent to:

Assigning weight w to the minority class in the loss function has the same effect as replicating minority samples w times — both increase attention to minority class errors.

Numerical Problems

▼
Problem 1 — Confusion Matrix Metrics
Given confusion matrix: [[TP=90, FP=10], [FN=5, TN=95]]. Compute Accuracy, Precision, Recall, and F1-score.
ValuesTP=90, FP=10, FN=5, TN=95. Total = 200.
Accuracy$(TP+TN)/\text{Total} = (90+95)/200 = 185/200 = 0.925 = 92.5\%$
Precision$TP/(TP+FP) = 90/(90+10) = 90/100 = 0.90$
Recall$TP/(TP+FN) = 90/(90+5) = 90/95 \approx 0.947$
F1$F1 = 2 \times 0.90 \times 0.947 / (0.90 + 0.947) = 1.7046/1.847 \approx 0.923$
Problem 2 — The Accuracy Trap
Dataset: 990 negative examples, 10 positive examples. Model predicts ALL as negative. Compute accuracy, precision, recall, and F1.
ConfusionTP=0 (missed all positives), FP=0 (never predicted positive), FN=10 (all 10 positives missed), TN=990.
Accuracy$(0+990)/1000 = 99\%$ — looks great!
Precision$TP/(TP+FP) = 0/(0+0) = \text{undefined}$ (0/0 — never predicted positive)
Recall$TP/(TP+FN) = 0/(0+10) = 0\%$ — caught ZERO of the 10 actual positives!
F1$F1 = 0$ (recall = 0 → harmonic mean = 0). The model is completely useless despite 99% accuracy.
Problem 3 — SMOTE Interpolation
Minority sample x_i = [2.0, 3.0]. Its nearest minority neighbor x_neighbor = [4.0, 5.0]. Random λ = 0.3. Compute the SMOTE-generated sample x_new.
Formula$x_{\text{new}} = x_i + \lambda(x_{\text{neighbor}} - x_i)$
Difference$x_{\text{neighbor}} - x_i = [4-2, 5-3] = [2, 2]$
Scaled$\lambda \times [2,2] = 0.3 \times [2,2] = [0.6, 0.6]$
Result$x_{\text{new}} = [2.0, 3.0] + [0.6, 0.6] = [2.6, 3.6]$. New synthetic minority sample at [2.6, 3.6].
